# 🎖️ Mark2 — 贾维斯中枢

> 项目状态：📐 设计完成，待部署
> 创建日期：2026-06-15
> 目标：从 Mark1（单机 4核8GB 虚拟机）迁移到独立的 8核32GB 云服务器，建成六层个人中枢

---

## 一句话

把贾维斯从公司虚拟机搬上独立云服务器，成为一个真正的个人中枢——连着所有设备、项目、服务，贾维斯坐在正中间。

---

## 架构总览

```
┌──────────────────────────────────────────────┐
│              🛡️ Cloudflare 边缘               │
│         Tunnel + Access + WAF (免费)          │
└──────────────────┬───────────────────────────┘
                   │
┌──────────────────▼───────────────────────────┐
│              ☁️ 贾维斯中枢 (Ubuntu 24.04)       │
│                                               │
│  L1 统一网关  ← Caddy (:443)                   │
│  L2 贾维斯核心 ← Gateway + embed + infos-handle│
│  L3 开发工作台 ← code-server + /srv/projects   │
│  L4 自部署服务 ← Portainer + Nextcloud + ...   │
│  L5 远程连接   ← Tailscale Mesh VPN            │
│  L6 数据备份   ← 每日 tar + Git + session-bk   │
│  L7 安全守护   ← ufw + fail2ban + auditd + ... │
└───────────────────────────────────────────────┘
```

## 加功能的标准流程

当用户说「Mark2项目 加个功能：

1. 在本文档「功能清单」里追加一条（编号 + 功能名 + 状态 = 待设计）
2. 判断功能落在哪一层（L1-L7）
3. 如果是已有层的新增：
   - 更新 `迁移方案 v3` 对应步骤
   - 如果有安全影响 → 更新 `安全体系设计`
   - 如果有新的依赖 → 更新 `部署启动手册` 依赖清单
4. 如果是全新的一层：
   - 在 `迁移方案 v3` 架构图新增该层
   - 在 `部署启动手册` 分层路线图新增该层
5. 改完后同步桌面 + git push
