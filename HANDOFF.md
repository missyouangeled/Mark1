## 当前系统状态（简版）

当前这套 `pulsenest-php/` 已经不只是论坛原型，而是在往可实际上线运营的论坛系统推进，已包含：

- 用户系统、登录注册、个人中心、公开用户主页
- 发帖 / 评论 / 回复 / 点赞 / 浏览量
- 帖子状态流：published / pending / hidden / draft
- 后台帖子审核队列与批量审核
- 举报系统：帖子 / 评论举报、后台举报队列、筛选、分页、批量处理、联动处置
- 通知闭环：审核结果、举报处理结果、内容状态变化
- 站点设置中心：登录 / 注册 / 举报 / 只读 / 审核 / 内容阈值
- 内容分发：最新 / 综合热度 / 最多回复 / 最多浏览 / 时间窗口热榜
- 首页运营模块：推荐作者、最高浏览、活跃版块、社区即时快照
- 创作者数据：累计获赞 / 浏览 / 回复 / 最近发帖
- 运营后台看板：新增量 / 积压量 / 活跃版块 / 活跃作者 / 举报分布 / 趋势
- 用户治理系统：治理记录、封禁联动停用、状态流转、高风险用户榜单
- staff 单用户治理档案页：`/user-governance.php?id=...`

## 最近 3 个最自然开发方向

如果用户没有指定下一步，就优先从下面 3 个方向里选一个继续：

1. **首页 / 前台产品化细修**
   - 统一板块间距与页面节奏
   - 首页中段与右侧栏继续微调
   - 列表 / 详情 / 用户页的视觉一致性继续收口

2. **staff 工作台继续增强**
   - `user-governance.php` 补更多就地操作
   - 从治理档案页直达后台对应帖子 / 评论 / 举报位置
   - 用户治理与内容治理之间继续打通

3. **运营后台继续数据化**
   - 更完整的趋势 / 排行 / 风险分布
   - 高风险用户、活跃版块、举报类型等维度继续增强
   - 让后台更接近真实运营看板

默认优先级判断：
- 用户如果正在盯页面效果 → 优先做前台产品化细修
- 用户如果正在盯后台管理 / staff 使用流程 → 优先做工作台增强
- 用户如果在问运营、数据、治理效率 → 优先做后台数据化
