# RULES_INDEX.md — 规则总索引（任何模型必须先读）

> 设计目标：极小、自解释、任何上下文窗口都能读全。
> 所有详细规则在 `rules/` 子目录，按域分层。本索引只做路由。

## 读取顺序（启动时）

1. **本文件**（RULES_INDEX.md）→ 必须读完
2. **按当前对话类型激活域文件**：

| 你在做什么 | 必须读 |
|-----------|--------|
| 日常聊天 / 陪伴 / 语音 / 千千 | `rules/chat.md` |
| 工作任务（代码/脚本/修改/项目/下载/排查） | `rules/work.md` |
| 系统操作（配置/补丁/升级/服务/磁盘/健康） | `rules/system.md` + `rules/work.md` |
| CPU 负载过高 / 系统卡顿 / 临时处理 | `docs/通用-CPU负载过高临时处理方案.md`（叠加在系统操作之上） |
| 任何涉及权限/外部操作/高风险动作 | `rules/safety.md`（叠加在上述域之上） |

3. **跨域叠加**：工作+聊天同时出现 → 都读。不确定 → 默认全读域摘要。

## 域文件结构（每个 ≤ 150 行）

| 文件 | 内容 | 大小上限 |
|------|------|----------|
| `rules/work.md` | 补丁流程、变更流水、分批操作规则、Git 提交、项目索引 | 150 行 |
| `rules/chat.md` | 语言强制中文、语音参数、千千 persona、陪伴风格、措辞偏好 | 150 行 |
