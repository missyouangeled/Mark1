## 增量索引投入生产（11:48）

### 解决的问题
- 之前每次 memory 文件变化需要全量重建索引（5379 段，163s）
- embed-sidecar 在 systemd 下不稳定（Python 3.11 uv 构建的 selectors fd=-1 bug）

### 增量索引架构
- memory-embed-index.py 新增 `--incremental-view` 模式
- 按文件 hash 对比，只重新生成变化段落的 embedding
- 通过 sidecar 的 `/encode` HTTP 接口获取向量（复用已加载的模型，避免 OOM）
- lifecycle-maintainer 每 5 分钟自动调一次增量检查
- 效果：163s → 15s（有变化）/ 0.1s（无变化），10x+ 提升

### embed-sidecar 稳定性修复
- 从 HTTPServer 改为原生 socket（完全绕过 Python 3.11 selectors bug）
- systemd 仍有兼容问题 → 使用 nohup 后台化 + cron 每 5min 健康检查兜底
- 搜索延迟稳定在 ~90ms，/encode 正常返回 dim=384

### 修改的文件
- 修改：scripts/embed-sidecar.py（原生 socket 重写）
- 修改：scripts/memory-embed-index.py（+增量索引 +sidecar 调用）
- 修改：scripts/openclaw-lifecycle-maintainer.py（+增量索引钩子）
- 新增：tools/embed-sidecar/embed-sidecar-wrapper.sh
