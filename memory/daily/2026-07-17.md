## 重要事项

### 1. Compaction 模型故障修复

**问题**：上下文过大时自动/手动 compaction 均失败。日志显示 `litellm/agnes-2.0-flash` 在 180s 超时内未能完成摘要（返回空内容）。

**修复**：
- `compaction.model`: `litellm/agnes-2.0-flash` -> `volcengine-agent/doubao-seed-2.0-pro`
- `compaction.memoryFlush.model`: 同上
- `compaction.timeoutSeconds`: 默认 180s -> 300s

**测试结果**（同等摘要任务）：
- agnes-2.0-flash: 11.31s + 17.12s，两轮均返回空内容
- doubao-seed-2.0-pro: 9.41s + 14.14s，两轮均返回高质量摘要

### 2. Mark42 Armor Guard 预警通道

**背景**：Mark42 Armor Guard 和 OpenClaw compaction 是两套独立系统。Armor Guard 检测到上下文过高时只写 JSON 文件，不通知对话层。

**改动**：`scripts/mark42_modules/armor.py` 新增 `_send_context_warn_event()` 函数 + 改造 `armor_guard()` 循环。

**链路**：
- 上下文 >= 70%（WARN）: 通过 `openclaw system event` 注入 systemEvent 到主会话，提醒模型主动 `/compact`
- 上下文 >= 85%（ALERT）: 自动触发 armor_compress + 注入预警
- 冷却 10 分钟防刷屏

**已测试**：`_send_context_warn_event(72.3)` 成功发送到主会话。
