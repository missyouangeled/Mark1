## 上午 (10:13-10:18) 修剩下 3 个 🟡

点点让 "修剩下 3 个 🟡 一并解决"。

### 🟡2: I 修复原子性 + fcntl 锁

**问题**: armor-guard / engine-daemon 多个进程并发 append broker, 裁的瞬间可能丢中间事件。
**修** (`scripts/mark42_modules/logs.py`):
- 加 `import fcntl`
- 用独立 .lock 文件, 避免锁住业务读路径
- LOCK_EX | LOCK_NB: 独占 + 非阻塞 (另一进程占锁就返 "跳过", 不死等)
- 锁范围覆盖 "读 size + 读 lines + 写 kept"
- 3 个新测试: test_concurrent_rotation_skipped_with_lock / test_lock_released_after_rotation / test_lock_path_separate_from_broker_file

### 🟡3: K 修复 4 Loop 异常 watchdog.log 留痕

**问题**: 4 Loop 挂掉时 notify_alert 推 dashboard, 但 watchdog.log 不记录 → 本地 audit gap。
**修** (`tools/mark42-watchdog/mark42-watchdog.sh`):
- 3 处 notify_alert 前都加 log()
  - loops-check-error → log "⚠️ 4 Loop 检查出错: $loops_check"
  - loops-missing → log "⚠️ 4 Loop 状态: $active/$total (expected: $total)"
  - loops-degraded → log "⚠️ Loop 状态不为 registered: $bad_loops"
- 1 处 loops 检查无输出 → log "⚠️ 4 Loop 检查无输出"

### 🟡4: J dry_run 模式 preBytes=null 加语义标记

**问题**: dry_run 写 actions.jsonl 时 preBytes=null, reader 困惑 "是没统计? 还是 dry_run 不写?"
**修** (`scripts/mark42_modules/armor.py`):
- 加 bytesStatus 字段, 4 个值:
  - `captured`: 压缩完成, 字段有真值
  - `skipped-dry-run`: dry_run 模式, 没真压缩, 字段为 null 是预期
  - `not-attempted`: 上下文 < THRESHOLD_WARN, 未尝试 compact
  - `error`: compact 报错 (没产生 postBytes)
- 2 个新测试: test_bytes_status_captured_on_successful_compact / test_bytes_status_skipped_dry_run
