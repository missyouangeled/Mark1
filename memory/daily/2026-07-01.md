## 心得

点点的 4 点批评**全部成立**, 而且比我意识到的更严重:

1. **"55% 是错觉"** — 这是我做"快速估算"时的常见 bias,把"代码写了"="完成"。**商品化维度的完成度** 要减:路径硬编码 / 文档缺失 / 安全隔离 / 真生产验证 / 核心覆盖。**真实数字 35%**,我之前算的 70% 是代码量完成度,不是商品化完成度。

2. **"13 天文档漂移"** — 这就是我自己写的"自动行为必须有据可查"的反例。**我在脑子里记"都修完了",但没写进文件**,差点让点点以为项目卡在 55%。下次"完成"时:**先写文档,再回话**。

3. **腐烂发现得晚 = 没人监控** — `mark42-tests.py` 走 subprocess,不在 pytest 套件,**没人每周跑**。**集成测试必须进 pytest**,或者**每周手动跑一次**。

4. **AI 的"完成"判定不可信** — 我之前说"6/30 全部收官"是 commit 视角,**不是商品化视角**。**商品化视角的"完成" = 别人能装、能跑、能解决他的问题**。当前 0 满足。
