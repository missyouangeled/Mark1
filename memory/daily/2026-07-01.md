## §21 — 13:36~13:39 收尾修口径：watchdog 双写 + memory-embed-search 自动切 venv

### 1) watchdog 同一条告警为什么会写两次
实查 `tools/mark42-watchdog/mark42-watchdog.sh` + `~/.config/systemd/user/mark42-watchdog.service` 后确认：
- 脚本里的 `log()` 使用了 `tee -a "$LOGFILE"`
- systemd unit 同时配置了：
  - `StandardOutput=append:/home/missyouangeled/.local/state/openclaw/mark42/watchdog.log`
  - `StandardError=append:/home/missyouangeled/.local/state/openclaw/mark42/watchdog.log`
- 结果：同一条 `echo` 被脚本 append 一次，systemd 再 append 一次 → `watchdog.log` 每条重复 2 次

**修法**：
- 给 `log()` / `run_and_log()` 增加 `tty` 判断
- 在 systemd 非 TTY 场景下，只往 stdout/stderr 写，由 unit 自己 append
- 在手动终端执行脚本时，仍保留 `tee -a` 便于现场查看

### 2) memory-embed-search 运行口径补齐
原问题：
- `python3 scripts/memory-embed-search.py 'Mark42 启动'`
- 在系统 python 下会报 `ModuleNotFoundError: numpy`
- 实际正确运行环境一直是 `~/.local/share/openclaw-embed-venv311/bin/python3`

**修法**：
- `scripts/memory-embed-search.py` 顶部改成：若缺 `numpy`，自动重执行到 embed venv
- 同时把 `query` 参数从 `nargs='+'` 改成 `nargs='*'`，让 `--check` 不再被“缺少 query”卡住
