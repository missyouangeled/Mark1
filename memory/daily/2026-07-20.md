- 升级成功：`2026.7.1 (2d2ddc4)` -> `2026.7.1-2 (0790d9f)`

**发现并修复的问题：**

1. **Branding 脚本失败（ExecStartPre exit 1）**
   - 根因：v2026.7.1-2 前端改用 Vite 代码分割，聊天逻辑从 `index-*.js` 拆到独立 `chat-page-*.js` chunk；所有函数名全变，6 个历史版本适配都匹配不上
   - 修复：在 `scripts/apply-openclaw-control-ui-branding.py` 中新增 v2026.7.1 版本适配分支
     - 文件扫描新增 `chat-page-*.js` glob
     - 函数映射：Bl=isHidden, ql=mergeHistory, O=roleNormalize, G=textExtract, ps=stripPrefix, Lb=unwrap, Pi=shouldSkip, Rb=isSending
     - 注入 Jarvis helper 函数 + yielded history replay + reading indicator
     - "无效重载" bug 已被新版本原生修复，不再需要补丁
   - 修复后 ExecStartPre 退出 0 ✅

2. **Gateway 重启卡在 deactivating**
   - 原因：升级时 gateway 内运行中的会话/任务阻止优雅退出
   - 处理：`systemctl --user kill --signal=SIGKILL` 强制终止后正常重启

3. **memory_search 超时**
   - 原因：embed-sidecar 在升级前已启动，与升级后的 gateway 连接异常
   - 修复：重启 `openclaw-embed-sidecar.service` 后恢复正常
   - 验证：memory_search 搜索 3.5s 返回 6 条结果 ✅

4. **MiniMax catalog schema 错误**
   - 原因：`openclaw.json` 中 MiniMax-M3 的 `input` 字段包含 `"video"`，新版本 schema 不允许
   - 修复：从 `openclaw.json` 的 minimax provider 配置中移除 `"video"`，删除旧 catalog.json 让 doctor 重新生成
   - 验证：doctor 不再报 schema 错误 ✅
