### Props 重命名规则（最终版，由美工 2026-06-04 下午重新确认）

**基础命名规则：**
- `.mat` 材质球 → 加前缀 `Mat_`
- `.fbx` 模型 / `.prefab` 预制件 / 贴图 → 加前缀 `Props_`
- 重名时加 `_a`/`_b`/`_c`（按字母排列）
- PascalCase：拼接单词首字母大写（如 `Brushwood` → `BrushWood`）
- 删除名称中的 `Type-`
- 删除 `-` 及后面的内容（如 `Box_06-WoodenCrate` → `Box_06`）

**名称精简规则：**
- 只保留有实际意义的单词，删除无意义字符/后缀
- `Stacks_of_boxes_YXY241212` → `Stacks_of_boxes`（删 `_YXY241212`）
- `Realistic_Viking_Fruit_Tent_Set` → `Tent`（只留核心词）
- `0_PropsGroups` → `PropsGroups`（去数字前缀）
- `Container03`/`Trough01` 等有意义的数字编号保留不变
- `Field_Kitchen_WW2_YXY241231` → `Field_Kitchen`（去无意义后缀）

**文件夹结构规则：**
- 每个 prefab → 独立文件夹（以 prefab 最终名字命名）
- 文件夹内三个子文件夹：`Prefab/`、`FBX/`、`Materials/`
- prefab 文件放 `Prefab/`，模型 fbx 放 `FBX/`，材质球 .mat 和贴图放 `Materials/`
- 多个 fbx 都放同一个 `FBX/` 下
- 共享材质放第一个使用它的 prefab 的 `Materials/` 下，不重复放
- .meta 文件始终同步重命名/移动

**特殊情况：**
- Pack_Medieval 不再特殊处理，全部按新规则统一
- 若整个名字就是物体名且拆分不出核心词，整体保留但做 PascalCase（如 `Brushwood` → `BrushWood`）
- 确定不了有没有意义的标 ⚠️待确认
