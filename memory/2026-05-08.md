## 2026-05-08 OpenClaw NVIDIA 音频接入进展（预压缩保存）

- 用户当前核心诉求：想在 OpenClaw 里用 NVIDIA 官方免费语音相关模型，不拘泥于实现方式，但原始诉求起点是“把 gateway / inference gateway 的音频端点转发接起来”。
- 已确认的关键事实：
  - 仅把 gateway 增加 `/v1/audio/speech`、`/v1/audio/transcriptions` 路由并不足以直接打通 NVIDIA 公共免费语音模型；因为这批公开模型主要走 `grpc.nvcf.nvidia.com:443` + `function-id` 的 NVCF / Riva 接口，而不是先前尝试的 `https://integrate.api.nvidia.com/v1/audio/...` OpenAI 兼容 REST 路径。
  - 适合的可行方案是：本机起一个本地 bridge，把 OpenAI 兼容音频端点转换到 NVIDIA Riva / NVCF gRPC，再由 OpenClaw gateway 代理到这个 bridge；或者至少先让 OpenClaw 内部音频能力走这个 bridge。
- 已在 workspace 内准备的文件（已写入）：
  - `tools/nvidia-audio-bridge/bridge.py`
    - 目标：提供本地 OpenAI 兼容的 `/v1/audio/speech` 与 `/v1/audio/transcriptions`。
    - 设计：读取 `~/.openclaw/openclaw.json` 中 `providers.nvidia.apiKey`；通过 `https://api.nvcf.nvidia.com/v2/nvcf/functions?visibility=public` 动态解析公共函数 id；再用 `nvidia-riva-client` 访问 `grpc.nvcf.nvidia.com:443`。
    - 当前支持的目标模型映射：
      - TTS：`nvidia/magpie-tts-multilingual`
      - ASR：`nvidia/parakeet-1_1b-rnnt-multilingual-asr`、`nvidia/parakeet-ctc-1_1b-asr`、`nvidia/parakeet-ctc-0_6b-zh-cn`、`nvidia/parakeet-ctc-0_6b-zh-tw`、`nvidia/canary-1b-asr`、`nvidia/whisper-large-v3`
    - 设计里包含：TTS 的 PCM→WAV/MP3 转换、上传音频经 ffmpeg 归一化后再送 ASR。
  - `scripts/apply-openclaw-nvidia-audio-gateway-patch.py`
    - 目标：给已安装的 OpenClaw `dist/server.impl-*.js` 打补丁，新增 gateway 对 `/v1/audio/speech` 与 `/v1/audio/transcriptions` 的处理，并写入 `openai-audio-http-nvidia-bridge.js` 代理模块，把请求转发到本地 bridge（默认 `http://127.0.0.1:18890`）。
- 已得到但尚未在本轮 flush 中执行验证的外部技术结论（来自压缩前上下文，应视为后续继续工作的依据）：
  - 通过 Python + `nvidia-riva-client` 直连 NVCF/Riva 已经打通了一条 TTS 调用路径，说明免费 TTS 至少是可接的。
  - ASR 中 `parakeet-1_1b-rnnt-multilingual-asr` 已被验证可以成功转写英文测试音频；说明 bridge 路线是成立的。

  3. 启动 `bridge.py`，先本地验证：
     - `GET /health`
     - `POST /v1/audio/speech`
     - `POST /v1/audio/transcriptions`
  4. 执行 `scripts/apply-openclaw-nvidia-audio-gateway-patch.py` 给本机 OpenClaw 安装产物打补丁。
