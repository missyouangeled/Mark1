- 当前剩余尾巴已从“链路不通”收敛为“首几条回复偏慢、`sendTyping` 偶尔超时但 `sendMessage` 仍成功”；后续若再排查，应优先盯首条消息启动慢、会话初次 processing 卡住、以及 typing 提示链路的稳定性，而不是重新从扫码/配对层面开始。
