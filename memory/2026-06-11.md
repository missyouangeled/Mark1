## 🐛 OpenClaw 升级 v5.22→v6.5：Control UI 黑屏问题排查与修复（17:00–17:36）

### 背景
点点从中午一直在折腾 OpenClaw 升级，v5.22→v6.5 后 Control UI 黑屏，已恢复 VM 快照 3 次。之前贾维斯两次说"修好了"但 Ctrl+Shift+R 又黑屏。

### 根本原因
品牌补丁脚本 `apply-openclaw-control-ui-branding.py` 里 `JARVIS_FUNCTIONS_V2026_6_5` 的 `JarvisShouldShowPendingReadingIndicator` 函数中，同一个 `for` 循环块内有两个 `let i` 声明：

```javascript
let i=uf(t);if(!i||!i.role)continue;...    // 第一次声明
let i=typeof t.timestamp=='number'?t.timestamp:0;...  // 重复声明 → SyntaxError
```

JS 严格模式不允许同作用域内 `let` 重复声明 → 整个 bundle 解析失败 → 白屏/黑屏。

### 次要问题
infos-handle 请求走 same-origin（port 18789），Gateway 不代理 `/v1/*` 路由，返回 HTML 导致 `JSON.parse: unexpected character`。

### 修复
1. `let i` 重声明 → `let ti`（commit `b7231be`）
2. infos-handle baseUrl 改为统一代理 port 18788（commit `abdd4db`）
3. Gateway 第一次重启后排水卡死，`kill -9` 强制重启

### 验证结果
- `node --check` 通过
- Control UI 初始加载 ✅
- Ctrl+Shift+R 硬刷新不黑屏 ✅  
- 聊天消息发送 ✅
- 品牌（贾维斯）注入正常 ✅
- Console 无 SyntaxError ✅

### 经验教训
1. 品牌补丁不能只看 HTTP 200 + grep，必须 `node --check` 做 JS 语法检查
2. 每次升级后必须浏览器实操硬刷新验证，不能只依赖后端状态
3. 品牌脚本里同一函数有多个版本（V22/V2026_6_5/NEW/CURRENT），修改时要确认实际走哪条链
4. brand 注入的去重检测逻辑（"Jarvis" 是否已在中间）会导致修了脚本但重打品牌时跳过——需要先还原原始 bundle
5. 后续升级 v6.x 时，注意品牌脚本里函数名映射是否变化（uf/w vs ij/ek vs MT/yT 等）
