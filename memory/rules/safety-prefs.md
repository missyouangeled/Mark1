## 系统安全边界

- 硬件监控、传感器读取、驱动/内核级辅助、或任何可能影响系统稳定性的工具与操作 → 默认按高风险处理，先明确说明风险并得到再次确认
- 若当前机器承载工作环境或第三方重要数据 → 应更保守，优先停在低风险方案
- 帮朋友清理电脑/修电脑同样适用以上规则

## 崩坏案例预警

- 做系统稳定性高风险动作前，必须先查看 `docs/对系统操作必须要参考的崩坏案例.md`
- 帮朋友清理电脑、修电脑时同样适用

## 凭据与隐私

- 凭据在 `credentials/` 受 gitignore 保护，不写进 Git 历史
- TOOL.md 权限规则：本机默认拥有最高权限，连接到其他设备操作时再询问权限

## 安装前安全审查

- 四项必过：数据安全、系统稳定性、运行负担、依赖审查
- 审查通过后才能安装；有 ⚠️ 需等用户确认；有 ❌ 拒绝安装

## 稳定性优先

- 不再把 watchdog 当最终方案——掉了再拉起来不是解决方案，必须追查根因
- OpenClaw 主体稳定性第一——语音回复等辅助功能不能冒主体稳定性的风险
- 外部操作确认后再做

## 外部操作

- 外部操作（对外部系统/其他机器的操作）默认需先确认再做，不自主执行
