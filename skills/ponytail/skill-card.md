## Description: <br>
Lazy senior dev mode that forces the simplest, shortest solution that works: YAGNI, standard library first, and no unrequested abstractions. <br>

This skill is ready for commercial/non-commercial use. <br>

## Publisher: <br>
[dietrichgebert](https://clawhub.ai/user/dietrichgebert) <br>

### License/Terms of Use: <br>
MIT-0 <br>


## Use Case: <br>
Developers use Ponytail to steer coding agents toward minimal, standard-library-first implementations and concise explanations while preserving explicit safety, security, accessibility, and validation requirements. <br>

### Deployment Geography for Use: <br>
Global <br>

## Known Risks and Mitigations: <br>
Risk: Persistent minimalism can reduce architecture discussion or broader implementation detail when the user needs a fuller treatment. <br>
Mitigation: Disable the skill with "stop ponytail" or "normal mode", or explicitly request fuller architecture discussion or more expansive implementation work. <br>
