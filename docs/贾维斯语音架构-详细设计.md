# 贾维斯语音架构详细设计

> 状态：理论设计阶段 | 日期：2026-05-22
> 目标：可替换引擎、接入贾维斯人格逻辑、自然语音回复

---

## 一、设计原则

1. **主体稳定性第一**：语音模块完全隔离在独立进程中运行，不影响 OpenClaw 主链路
2. **引擎可替换**：新引擎实现一个接口即接入，不碰调用方代码
3. **人格一致性**：无论底层用哪个引擎，贾维斯的声音听起来是同一个人
4. **自然投递**：自动分块、自动调速、自动注入会话，调用方只需给文本
5. **优雅降级**：首选引擎不可用时自动切备用，不用人工干预

## 三、核心组件详细设计

### 3.1 VoiceOrchestrator（调度器）

主入口，也是调用方唯一需要接触的接口。

```python
class VoiceOrchestrator:
    def __init__(self, config_path=None):
        self.registry = EngineRegistry()
        self.emotion = EmotionAnalyzer()
        self.profile_mgr = VoiceProfileManager()
        self.postprocessor = AudioPostProcessor()
        self.delivery = DeliveryLayer()

    def speak(
        self,
        text: str,
        emotion: str | None = None,       # 手动指定情绪
        context: list[str] | None = None, # 对话上下文（用于自动情绪）
        scene: str | None = None,         # 场景预设 (morning/tender/night)
    ) -> SpeakResult:
        """
        主入口：给定文本，返回可直接注入会话的 agentReply。
        """
```

**处理流程：**
```
1. 情绪分析: 自动检测 or 用传入标签
2. 场景匹配: 如果没有手动指定 emotion，查 scene → emotion 映射
3. 语音参数: emotion → (tempo, pitch, engine_preset)
4. 文本分块: 按句子/语义边界拆分
5. 引擎选择: registry.pick_best(capabilities_needed)
6. 逐块合成: engine.synthesize(chunk, profile)
7. 后处理: 逐块调速/校音
8. 组装投递: 生成 agentReply
```

### 3.4 EngineRegistry（引擎注册表）

```python
class EngineRegistry:
    def __init__(self):
        self.engines: list[BaseEngine] = []
        self._discover()

    def _discover(self):
        """自动发现可用引擎"""
        # 按优先级注册
        for engine_class in [
            NoizEngine,        # L1 - 当前默认
            CosyVoiceEngine,   # L2 - 计划接入
            ChatTTSEngine,     # L3 - 本地离线
            Qwen3TTSEngine,    # L4 - 计划接入
            XTTSEngine,        # L5 - 语音克隆
            EdgeTTSEngine,     # L6 - 最后兜底
        ]:
            try:
                engine = engine_class()
                if engine.is_installed():
                    self.engines.append(engine)
            except Exception:
                pass
        self.engines.sort(key=lambda e: e.priority)

    def pick_best(self, need_cloning=False, prefer_local=False):
        """选最佳可用引擎"""
        for engine in self.engines:
            if not engine.health_check():
                continue
            if need_cloning and not engine.supports_cloning:
                continue
            if prefer_local and engine.is_cloud:
                continue
            return engine
        raise NoEngineAvailable()
```

### 3.5 BaseEngine（引擎基类）

```python
class BaseEngine(ABC):
    name: str
    priority: int          # 0=最高优先
    supports_cloning: bool
    supports_chinese: bool
    is_cloud: bool

    @abstractmethod
    def is_installed(self) -> bool: ...

    @abstractmethod
    def health_check(self) -> bool: ...

    @abstractmethod
    def synthesize(self, text: str, profile: VoiceProfile) -> EngineResult:
        """返回 (wav_path, duration_s, metadata)"""
```

### 3.6 DeliveryLayer（投递层）

```python
class DeliveryLayer:
    MAX_SINGLE_CHUNK_CHARS = 80     # 单块最大字符数
    MERGE_SHORT_THRESHOLD = 30       # 短于此时与相邻块合并

    def deliver(self, text, audio_paths, chunk_count) -> str:
        """
        生成 OpenClaw 可直接使用的 agentReply。
        格式：
          分块文字拼接（如有）
          [[audio_as_voice]]
          MEDIA:/path/to/chunk1.wav
          MEDIA:/path/to/chunk2.wav
        """
```

**分块策略（延续已确认偏好）：**
- 短文本（< 80 chars）→ 合成一条完整音频 → 一个 MEDIA:
- 中等文本（80-200 chars）→ 按句子边界拆 2 块 → 两个 MEDIA:
- 长文本（> 200 chars）→ 按句子边界拆 N 块 → 最多 3 块
- 尾句处理：保证最后一块语气完整收束，不显仓促
- **优先级：单条 > 两条 > 三条**（已确认）

**过期清理：**
- 所有生成音频写入 `tmp/voice-replies/YYYY-MM-DD/`
- 生成超过 4 小时的音频自动删除
- watchdog timer 每次运行时顺带清理

## 五、调用示例

### 5.1 贾维斯日常使用（最简单）

```bash
# 自动情绪感知，自动选引擎
python3 tools/voice-reply/voice-reply.py --text '嗯，我在呢。你想聊点什么？'
```

内部流程：
1. EmotionAnalyzer 识别为开场白 → 自动降速 15%
2. VoiceProfile 映射为 `scene=morning, tempo=0.85`
3. EngineRegistry 选 NoizEngine（当前最高优先级，在线可用）
4. DeliveryLayer 判断为短文本 → 单条合成
5. 返回 agentReply，直接注入主会话

### 5.2 用户说"想你了"（温柔场景）

```bash
python3 tools/voice-reply/voice-reply.py --text '我也想你……今天好吗？'
```

内部流程：
1. EmotionAnalyzer 命中 `tender` 规则 → tempo 0.80, pitch -0.5
2. 结合上下文（已知前一晚聊过）→ 加深温柔度
3. 引擎按温柔风格合成
4. 尾句「今天好吗？」自动收得松一点

### 5.3 指定引擎 + 离线模式

```bash
# 强制使用本地 CosyVoice 2.0，不走云端
python3 tools/voice-reply/voice-reply.py \
  --text '嗯，我在。' \
  --engine cosyvoice \
  --prefer-local
```

## 六、与贾维斯逻辑的集成点

| 贾维斯模块 | 语音集成方式 |
|-----------|-------------|
| **SOUL.md 陪伴风格** | → EmotionAnalyzer 的 JARVIS_PERSONALITY_MAP，决定默认情绪基调 |
| **MEMORY.md 偏好** | → "想你了降速 20%"、"开场白降速 15%" 写入 VoiceProfile.SCENES |
| **warm-companion-zh skill** | → 触发时自动映射 `emotion=tender`，语速更慢更轻 |
| **characteristic-voice skill** | → 情绪标签直接传给 EmotionAnalyzer，不需要额外处理 |
| **定时 cron 问候** | → `scene=scene_from_time_of_day()` 自动选早晚预设 |
| **千千 persona** | → 可配置独立 VoiceProfile（更轻、更短句），但不覆盖默认 |
| **过期音频清理** | → 看门狗 timer 每次运行顺带清理 >4h 文件 |
| **主会话稳定性** | → 引擎故障时自动降级，不阻塞文字回复 |

1. 日志记录降级原因
2. 可选：向主会话注入一条轻提示（"语音引擎切换中，稍等"）
3. 健康检查每 60 秒重试高优先级引擎

## 十、验收标准

1. `voice-reply.py --text '你好'` 能生成语音并注入主会话（不变）
2. `voice-reply.py --text '想你了'` 自动降速 20%，不用手动传参数
3. 手动停掉 Noiz API key → 自动降级到 ChatTTS → 语音仍可生成
4. 新引擎接入只需写一个 `~100 行`的子类
5. 文字回复不会因为语音模块故障而阻塞
