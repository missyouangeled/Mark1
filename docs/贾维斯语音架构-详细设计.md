# 贾维斯语音架构详细设计 v2

> 状态：理论设计阶段 | 日期：2026-05-22
> 目标：可替换引擎、接入贾维斯人格、自然语音回复
> v2 变更：6层缩为3层，移除不实承诺（统一声线/上下文感知），增加缓存和批量合成

---

## 一、设计原则

1. **主体稳定性第一**：语音模块独立运行，不影响 OpenClaw 主链路
2. **引擎可替换**：新引擎实现一个接口即接入
3. **人格感知**：根据场景和文本浅层特征自动匹配语音参数
4. **自然投递**：分块 → 合成 → 后处理 → 返回 agentReply
5. **优雅降级**：引擎不可用时自动切备用，含 60s 缓存避免重复检查

---

## 三、核心组件

### 3.1 VoiceOrchestrator（调度器）

```python
class VoiceOrchestrator:
    """单入口。给定文本，返回可直接注入会话的 agentReply。"""

    def __init__(self):
        self.registry = EngineRegistry()
        self.scenes = ScenePresets()
        self.postprocessor = AudioPostProcessor()

    def speak(
        self,
        text: str,
        scene: str | None = None,       # "morning"|"tender"|"night"|"warm"
        prefer_local: bool = False,
    ) -> SpeakResult:
        """
        处理流程:
          1. 场景 → 语音参数 (tempo/pitch/style)
          2. 关键词浅层辅助微调
          3. 引擎选择 (cached health + 优先级)
          4. 文本分块
          5. 批量合成 (同一引擎)
          6. 逐块后处理
          7. 组装 agentReply
        """
```

### 3.2 ScenePresets（场景预设）

```python
class ScenePresets:
    """场景 → 语音参数。纯映射，不做重活。"""

    PRESETS = {
        "morning": {"tempo": 0.85, "style": "warm"},
        "tender":  {"tempo": 0.80, "pitch_semitones": -0.5, "style": "gentle"},
        "night":   {"tempo": 0.78, "pitch_semitones": -1.5, "style": "late-night"},
        "warm":    {"tempo": 1.05, "style": "natural"},
    }

    # 浅层关键词微调（只做确定性能做的）
    KEYWORD_TWEAKS = [
        (["想你", "抱抱", "孤单", "在吗"], {"tempo_bias": -0.05}),   # 温柔词 → 再降 5%
        (["哈哈", "厉害", "666"],           {"tempo_bias": +0.05}),   # 兴奋词 → 再升 5%
        (["晚安", "睡"],                     {"tempo_bias": -0.05}),   # 睡前词 → 再降 5%
    ]

    def resolve(self, text: str, scene: str | None = None) -> VoiceProfile:
        """场景预设 + 关键词微调 → VoiceProfile"""
```

### 3.3 EngineRegistry（引擎注册表）

```python
class EngineRegistry:
    def __init__(self):
        self.engines: list[BaseEngine] = []
        self._health_cache: dict[str, HealthCacheEntry] = {}
        self.HEALTH_CACHE_S = 60           # 健康检查缓存
        self.HEALTH_CHECK_TIMEOUT_S = 3    # 单次检查超时

    def pick_best(self, need_cloning=False, prefer_local=False) -> BaseEngine:
        """
        按优先级遍历引擎:
          1. 查缓存（60s 内有效）
          2. 缓存过时 → 轻量重检（文件存在/进程可访问，不真跑合成）
          3. 选第一个健康的
          4. 全部不可用 → 降级继续 or 抛异常
        """
```

### 3.4 BaseEngine（引擎基类）

```python
class BaseEngine(ABC):
    name: str           # "noiz" / "cosyvoice" / "chattts" / ...
    priority: int       # 0 = 最高优先
    is_cloud: bool

    @abstractmethod
    def is_installed(self) -> bool:
        """检查依赖是否存在（不跑合成，纯文件/命令检查）"""
        ...

    @abstractmethod
    def health_check(self) -> bool:
        """轻量检查：依赖可访问、环境正常。不真跑合成。"""
        ...

    @abstractmethod
    def synthesize_all(
        self, texts: list[str], profile: VoiceProfile
    ) -> list[EngineResult]:
        """批量合成。同一引擎一次处理全部分块，保证音色一致。失败抛异常。"""
        ...

    def synthesize_one(self, text: str, profile: VoiceProfile) -> EngineResult:
        """单条合成（内部调 synthesize_all）"""
        return self.synthesize_all([text], profile)[0]
```

**引擎实现：**

| 引擎 | 定位 | 实际做的事 |
|------|------|-----------|
| NoizEngine | L1 云端 | 调 `noiz-reply.sh --style {profile.style} --pitch-semitones {profile.pitch}` |
| CosyVoiceEngine | L2 本地 (计划) | 独立 venv，pip 安装 CosyVoice 2.0，用 ref_audio 做零样本克隆 |
| ChatTTSEngine | L3 本地离线 | 调 `chattts-stable.sh --preset {profile.style}` |
| XTTSEngine | L4 语音克隆 | 调 `local-xtts-reply.sh` |
| EdgeTTSEngine | L5 最终兜底 | 调 `tts.mjs --voice zh-CN-XiaoxiaoNeural` |

### 3.5 AudioPostProcessor（后处理）

```python
class AudioPostProcessor:
    """后处理管道。纯 ffmpeg 调用，无模型依赖。"""

    def process(self, audio_path: str, profile: VoiceProfile) -> str:
        """输入原始音频 → 输出后处理后的 wav 路径"""

    # 管道:
    # 1. 格式统一 → wav
    # 2. tempo 调整 → rubberband (保留音色)
    # 3. pitch 校正 → rubberband formant=preserved
    # 4. 静音裁剪 → silenceremove
    # 5. 响度归一化 → loudnorm
```

### 3.6 投递

```python
def build_agent_reply(text: str, audio_paths: list[str]) -> str:
    """
    返回可直接注入 OpenClaw 会话的字符串。
    格式:
      {原始文本}
      [[audio_as_voice]]
      MEDIA:/path/to/chunk1.wav
      MEDIA:/path/to/chunk2.wav

```

**注入由调用方负责。** `voice-reply.py` 不耦合注入逻辑。调用方拿到 agentReply 后自己 `chat.inject`。

## 四、调用场景

### 场景 A：贾维斯回复语音（最常见）

```bash
# Agent 生成回复文本后调用
python3 tools/voice-reply/voice-reply.py \
  --text '嗯，我在呢。你想聊点什么？' \
  --scene morning
```
→ 自动 tempo 0.85、warm 风格、选 Noiz 引擎、单条合成 → agentReply

### 场景 B：用户说"想你了"

```bash
python3 tools/voice-reply/voice-reply.py \
  --text '我也想你……今天好吗？' \
  --scene tender
```
→ tender 预设 tempo 0.80,pitch -0.5 + 关键词"想你"再降 5% = **tempo 0.75**

### 场景 C：离线模式（强制本地）

```bash
python3 tools/voice-reply/voice-reply.py \
  --text '你好' \
  --prefer-local
```
→ 跳过 Noiz，直接选 ChatTTS 或 CosyVoice

### 场景 D：Agent 集成

```python
# 在主 agent 回复逻辑中
result = subprocess.run([
    "python3", "tools/voice-reply/voice-reply.py",
    "--text", reply_text,
    "--scene", detect_scene_from_text(reply_text),
    "--json-output",
], capture_output=True, text=True)

data = json.loads(result.stdout)
# data["agentReply"] → 直接当作 assistant 回复内容返回
```

## 六、降级链路

```
Noiz 云端 (L1)
  ↓ 不可用 / --prefer-local
CosyVoice 本地 (L2)  ← 计划接入，当前跳过
  ↓ 不可用
ChatTTS 本地 (L3)
  ↓ 不可用
msedge-tts (L4)
  ↓ 不可用
→ 纯文本（不带语音）+ 日志记录
```

- health_check 缓存 60s，单个引擎检查超时 3s
- 降级时记录原因到日志，不阻断文字回复

## 七、与贾维斯逻辑的集成

| 贾维斯模块 | 集成方式 |
|-----------|---------|
| **场景选择** | cron 问候 → `--scene morning/night`；warm-companion-zh → `--scene tender` |
| **默认声线** | 所有引擎预设锁定至用户确认的最佳音色 (zh-hybrid-default-template) |
| **语速偏好** | tender 预设 tempo 0.80（已含 MEMORY.md 的降速 20% 要求） |
| **开场白降速** | morning 预设 tempo 0.85（已含 15% 降速） |
| **千千 persona** | 独立 ScenePreset，更轻声更短句（后续扩展） |
| **过期清理** | 看门狗 timer 顺带清理 >4h 音频 |
| **稳定性** | 引擎故障 → 降级 → 纯文本，不阻塞主回复 |

## 八、不做什么（明确边界）

| 不做 | 原因 |
|------|------|
| ❌ 跨引擎统一声线 | 模型结构不同，同一 ref_audio 输出音色差异大，架构层解决不了 |
| ❌ 对话上下文感知 | 每次独立调用无历史，做了也是假的。等有状态引擎时再谈 |
| ❌ 引擎健康检查跑真合成 | 太慢。纯文件/命令检查，3s 超时 |
| ❌ 分块用不同引擎 | 音色不统一。整批绑定同一引擎 |
| ❌ voice-reply.py 负责注入 | 解耦。只返回字符串，调用方注入 |

## 十、验收标准

1. `voice-reply.py --text '你好' --scene warm` → 生成语音，注入主会话可播放
2. `voice-reply.py --text '想你了' --scene tender` → tempo 自动降至 0.75-0.80，不需要手动传参数
3. 停掉 Noiz API key → 自动降级到 ChatTTS → 语音正常生成（延迟略增）
4. 新引擎接入只需写一个 `~80 行` 的子类（实现 `is_installed` / `health_check` / `synthesize_all`）
5. 语音模块故障时，文字回复照常发送（不受影响）
