- **Minimax Token Plan**：实际 baseUrl 是 `https://api.minimax.chat/v1`（不是 `api.minimax.io`），Token Plan 的 Subscription Key 形如 `sk-cp-...` 在该端点正常鉴权。
- **Caddy X-Real-IP 行为**：Caddy 的 `header_up X-Real-IP {remote_host}` 在 reverse_proxy 内部 `remote_host` 是 Caddy 视角的连接端 IP，**会用 Caddy 视角覆盖上游传过来的 X-Real-IP**。验证鉴权需要从 LAN IP 真实访问而非用 X-Forwarded-For 头（会被 Caddy 覆盖）。
