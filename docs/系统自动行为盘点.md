# 系统自动行为盘点

> **维护原则**：点点原则 —— "所有自动 / 不可见行为必须有据可查"（见 `MEMORY.md`）
>
> **盘点目的**：当某自动行为坏了、出错了、或产生了"我看不到的副作用"时，**知道去查哪里**、**怎么修**
>
> **盘点日期**：2026-07-06（首次盘点）
>
> **盘点人**：贾维斯

---

## 0. 盘点方法

每个行为都问这 4 个问题：

1. **触发机制**：什么时候跑（定时 / 事件 / 手动）
2. **执行内容**：它具体做什么
3. **留痕位置**：跑完之后能看到什么（日志 / 状态文件 / 注入消息）
4. **失败信号**：出错了能在哪里看到（journald / 文件 / 注入主会话）

## 2. OpenClaw 自带 watcher / daemon（救命 1 之外的 session 守护）

**重要认识**：OpenClaw **自带 session 大小监控** —— **救命 1 之前存在的事实上重复造轮子**。本节梳理这些 daemon，明确**谁管什么**。

### 2.1 `openclaw-session-size-watcher`

**这是我之前漏看的救命 1 等价物** —— 比我的实现更稳。

| 项 | 值 |
|---|---|
| 类型 | systemd user timer + 事件驱动 |
| timer 频率 | 每 2 分钟（`OnUnitActiveSec=120s`）|
| 事件频率 | 每次主会话消息（`--gate-seconds 60` 去重）|
| 触发命令 | `python3 scripts/openclaw-session-size-watcher.py --systemd --print-json`|
| **功能** | 按总目录大小分层清理 |
| 阈值 | INFO（< 25MB）→ 记录；CRITICAL（> 25MB）→ 清理 checkpoint/trajectory/bak/reset；FORCE_CLEAN（> 40MB）→ 还清死会话 jsonl |
| 留痕 | `~/.local/state/openclaw/session-size-watcher/state.json`（最近 200 条历史）+ `alerts.json`（异常告警）|
| **关键特点** | **不动主 session jsonl**（只清理 ckpt/traj/bak），**不主动通知主会话** |
| 失败信号 | `alerts.json` 里有未读项（**需要主 agent / 其它 watcher 主动读这个文件**）|
| **历史数据** | 最近清理：2026-07-06 02:04:30，清掉 44 文件 / 23.23MB |

**问题**：**这个 watcher 有 alerts 但没人读**。事实佐证：当前 trajectory = 7.22MB > 阈值 5MB，**watcher 04:45 开始就在 alerts.json 写告警，但没注入主会话**。如果哪天主 agent 主动读 alerts.json，能救；如果不主动读，没人知道。

## 8. 当自动行为出问题时怎么查

### 救命类

| 现象 | 怎么查 |
|---|---|
| 主会话没有收到救命 1 告警 | `openclaw cron runs --id b560cd2c-...` |
| session-size-watcher 没清理 | `cat ~/.local/state/openclaw/session-size-watcher/state.json` 看 last_cleanup |
| gateway 资源超限 | `systemctl --user status openclaw-gateway` + `journalctl --user -u openclaw-gateway --since "5min ago"` |

### 非救命 daemon

| daemon | 状态文件 | 日志 |
|---|---|---|
| frontstage-guardian | `~/.local/state/openclaw/frontstage-guardian/last-report.json` | `guardian.log` |
| health-collector | `~/.local/state/openclaw/health-collector/` | (同目录) |
| task-scheduler | (同目录) | (同目录) |
| session-backup | `/mnt/data/openclaw/session-backup/backup-manifest.json` | (snapshots) |
| resume-watch | (自己路径) | (自己路径) |
| mark42-watchdog | `tools/mark42-watchdog/` | (自己路径) |
| lifecycle-maintainer | `~/.local/state/openclaw/lifecycle-maintainer/` | `maintainer.log` |

### 系统级

| 信号 | 查哪里 |
|---|---|
| gateway restart | `journalctl --user -u openclaw-gateway` |
| session 自动 corrupt | sessions 目录 `*.corrupt-*.jsonl` |
| 内存超限 | `systemctl --user status` 内存字段 |
