## 🖼️ 画图模型

### Agnes Image 2.0 Flash

- **API 端点**: `https://apihub.agnes-ai.com/v1/images/generations`
- **API Key 位置**: `~/.openclaw/openclaw.json` → `models.providers.litellm.apiKey`
- **接入方式**: LiteLLM 通道（`litellm/agnes-image-2.0-flash`）| 直接 HTTP API
- **费用**: 免费
- **注册**: https://apihub.agnes-ai.com

**已知问题**:
- ⚠️ LiteLLM 通道经常报 `malformed response`（litellm 与 Agnes 的响应格式不完全兼容，2026-06-12 首次发现，2026-06-16 复现 3 次）
- ⚠️ 直接 HTTP API 也可能超时（2026-06-16 curl 连接超时、exit code 28）
- ⚠️ 服务端间竭性不稳定，高峰期容易排队/无响应

**绕过方案**:
- ✅ **方法1（推荐）**: 绕过 LiteLLM，直接用 `curl`/`requests` 调原生 API
  ```bash
  curl -X POST "https://apihub.agnes-ai.com/v1/images/generations" \
    -H "Authorization: Bearer $AGNES_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"model":"agnes-image-2.0-flash","prompt":"...","n":1,"size":"1024x1024"}'
  ```
- ✅ **方法2**: 换用 Agnes Image 2.1 Flash（`litellm/agnes-image-2.1-flash`），但也可能遇到同样问题
- ⚠️ **备选**: 如果 Agnes 全线不可用，切 OpenAI DALL-E 路线（但需注意是否付费）

**使用记录**:
| 日期 | 模型 | 方式 | 结果 | 备注 |
|------|------|------|------|------|
| 2026-06-12 | 2.0-flash | 直接 API | ✅ 成功 | 生成 3 张图，10-20s/张 |
| 2026-06-12 | 2.0-flash | LiteLLM | ❌ malformed response | 同一天直调成功 |
| 2026-06-16 | 2.1-flash | LiteLLM | ❌ malformed response | — |
| 2026-06-16 | 2.0-flash | LiteLLM | ❌ HTTP 500 upstream error | — |
| 2026-06-16 | 2.0-flash | LiteLLM | ❌ malformed response | — |
| 2026-06-16 | 2.1-flash | LiteLLM | ❌ malformed response | — |
| 2026-06-16 | 2.0-flash | 直接 API curl | ❌ 超时 (exit 28) | 即时用 1K 分辨率也超时 |

**最佳实践**:
- 优先用直接 API（绕过 LiteLLM）
- 如果直调也超时，等 10 分钟再试
- 尽量用 1024x1024（方形）更快，非方形尺寸会慢
- 人物半身/面部特写最稳，全身+复杂姿态容易翻车

- **API 端点**: `https://api.minimaxi.com/v1/music_generation`
- **歌词生成**: `https://api.minimaxi.com/v1/lyrics_generation`
- **API Key**: `sk-api-WEsVwiK_4vgKRLas9hEFczIAbn3E9cwYorWaECbeYNMoOlOY_PCeS3rExYAdFwc6WpS84_ls0hzuVp7TbVhkac9CJ47ClBHpxmXLkZvqvGnfaTt_vGxIfOg`
- **文档**: https://platform.minimax.io/docs/guides/music-generation
- **官网体验**: https://www.minimax.io/audio
- **费用**: 2026-04-10 发布，14 天限免已过期（截至 2026-06-16 API 需充值）
- **官网免费额度**: 限免期间每日 500 首（普通用户），登录后官网仍可使用
- **特点**: 20 秒首包延迟、BPM/调性精准控制、Cover 翻唱功能、国风器乐理解好、中低频优化
- **关键参数**: `model: "music-2.6"`, `is_instrumental: true`（纯器乐）, `lyrics`（歌词）,
  `lyrics_optimizer: true`（无歌词时自动生成）
- **充值后可用**: API 余额不足（error 1008）时需在 MiniMax 平台充值

**🖥️ Playwright 自动登录方案**（绕过 API 付费，直接操作官网网页）:
- **原理**: Playwright 持久化 storage state — 真人登录一次，保存 cookie/localStorage，AI 后续全自动复用
- **环境**: `~/.local/share/playwright-venv`（Python 3.12 + Playwright 1.60.0 + Chromium）
- **登录脚本**: `python3 scripts/minimax-login-save.py`
  - 打开浏览器 → 手动登录 MiniMax → 按 Enter 保存 `~/.local/state/openclaw/minimax-storage-state.json`
- **生成脚本**: `python3 scripts/minimax-music-gen.py --prompt '风格描述' --lyrics '歌词'`
  - 自动复用登录态，填歌词+prompt+点生成
  - 支持 `--instrumental`（纯器乐）、`--lyrics-optimizer`（AI 自动填词）、`--song-name`、`--headless`（无窗口）
  - 首次使用前需先运行 `minimax-login-save.py` 登录一次
- **当前状态**: ✅ 已安装 Playwright + Chromium，脚本已就绪，待首次登录
