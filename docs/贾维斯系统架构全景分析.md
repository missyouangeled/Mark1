## 六、数据流关键路径

### 6.1 用户消息 → AI 回复（热路径）

```
用户输入 (Control UI / WebChat)
  → unified-proxy :18788 路由
  → Gateway :18789 接收
  → 模型路由 (deepseek-company/deepseek-v4-pro)
  → Tool 调用 (exec/read/write/search/...)
  → 生成回复
  → unified-proxy → 用户前端
  ⏱ 目标延迟: < 3s (简单问答)  /  < 30s (复杂工具链)
```

### 6.2 辅助消息管道（冷路径）

```
watcher 检测到事件 (dirty flag)
  → health-collector 写入 broker 事件
  → frontstage-broker 重建视图
  → infos-handle sidecar 提供 API
  → Control UI 品牌注入读取
  → 嵌入式仪表板更新

监工状态变化:
  supervisor-status.json 更新
  → frontstage-broker dirty flag
  → broker 重建 views
  → infos-handle /v1/query/snapshot.summary
  → Control UI 仪表板 "监工" 卡片刷新
```

### 6.3 记忆搜索路由 (L1→L2→L3→L4)

```
用户提问 → memory_search
  │
  ├─ L1: memory-search-local-first.py
  │     grep MEMORY.md + memory/*.md (0.1s)
  │     置信度 ≥0.7 → 短路返回 (60s TTL 缓存)
  │
  ├─ L2: memory-search-router.py
  │     MEMORY_INDEX.yaml 关键词匹配 (0.05s)
  │     QMD BM25 全文检索 (0.2-0.5s)
  │
  ├─ L3: cloud memory_search (向量语义, 4-10s)
  │
  └─ L4: session-backup context-summary (文件直读兜底)
```

## 八、健壮性机制

### 退避与自愈
- **frontstage-guardian**: 连续 4 次错误 → 停止检查 + 跳过 3 轮 → 下一轮健康恢复后重置
- **embed-sidecar**: cron 每 5min 健康检查，挂了自动 nohup 拉起
- **resume-watch**: boot_id 跳变 → 自动 `systemctl --user restart openclaw-gateway`
- **session-size-watcher**: CRITICAL=25MB → 清理旧 session / FORCE_CLEAN=40MB → 更强清理

### 升级后自检
- **触发**: BOOT.md → `openclaw-post-upgrade-self-check.py`
- **检查**: 版本号变化、关键补丁、broker、infos-handle、sidecar、unified-proxy、recovery watcher
- **兜底**: 自检失败不阻塞启动，记录到升级记录

### 黑屏应急
- **诊断**: `openclaw-control-ui-emergency.py --check` (L1→L4 分层检查)
- **修复**: `--repair` 低风险修复 / `--safe-mode` 临时禁用品牌注入兜底
- **预防**: GitHub 推送后等 5-10 秒再刷新 Control UI

## 十、架构总结

```
层级模型: 入口层 → 代理层 → 核心服务层 → 守护层 → 数据层

设计原则:
  1. 统一入口: 所有流量走 Caddy :18788 → 单一鉴权点
  2. 关注分离: Gateway(会话/模型) / Sidecar(数据) / Embed(向量) 各司其职
  3. 兜底多层: L1→L4 搜索 / 本地诊断 + cloud API / 数据盘备份 + workspace
  4. 自愈优先: 退避重试 / cron 保活 / resume 自动重启 / session 体积防膨胀
  5. 可观测性: 补丁 12 项验证 + broker 视图仪表板 + 变更流水全记录

当前规模:
  • 4 个常驻服务进程
  • 6 个 systemd timer (0 个 failed)
  • 5 个 cron 自动化任务
  • 12 项补丁检查 (12/12 ✅)
  • ~50 个 Skill
  • 记忆索引 7173 关键词
```
