# 多机器读取与更新规则

- 适用机器：通用
- 系统 / OS：通用
- 文档类型：跨机器共享规则

## 用途

这份文档定义：当 OpenClaw 在不同机器上启动、读取系统相关说明、或更新 GitHub 中的机器专用文档时，应该如何快速判断“当前在哪台机器上”、以及“当前应该优先看哪些内容、更新哪些内容”。

## 第一步：先判断当前机器

必须按下面顺序判断当前机器身份：

1. 运行时 `host` 元数据
2. 系统 hostname / computer name
3. 本地稳定 IP（仅作兜底）

判断依据统一来自：

- `HOST_CONTEXT.md`

禁止只凭“上一次在哪台机器”或模糊印象去猜。

## 第二步：得出当前环境标签与系统 / OS

识别出机器后，至少明确这两个值：

- 环境标签：例如 `公司` / `掌机` / `家里` / `未归类`
- 系统 / OS：例如 `Linux` / `Windows` / `macOS`

当前已知示例：

- `公司（Linux）`
- `掌机（Windows）`

## 第三步：读取系统相关文档时的优先级

当任务涉及以下内容时，都视为“系统相关文档 / 脚本 / 修复说明”：

- OpenClaw gateway
- 开机启动 / 保活 / watchdog
- systemd / Scheduled Task
- 本机脚本
- SSD / 电源 / 系统维护
- 机器专用故障修复

读取优先级必须按下面执行：

### 优先级 1：当前机器精确匹配的文档

优先读取正文中明确标注了当前环境和系统的文档，例如：

- 当前在公司 Linux 机 → 优先读 `适用机器：公司（Linux）`
- 当前在掌机 Windows → 优先读 `适用机器：掌机（Windows）`

### 优先级 2：通用文档

如果需要补充跨机器共用规则，再读：

- `适用机器：通用`
- `系统 / OS：通用`

### 优先级 3：其他机器专用文档仅作参考

如果文档标注的是其他机器，例如当前在公司 Linux，却读到 `掌机（Windows）`：

- 可以参考思路
- 但不能直接照抄执行步骤
- 更不能把 Windows 的脚本、计划任务、路径规则直接应用到 Linux

## 第四步：更新内容时的归类规则

### 如果修改只影响当前机器

必须：

1. 在正文写明 `适用机器`
2. 写明 `系统 / OS`
3. 写明用途和影响范围
4. 尽量把说明集中写进对应维护文档，而不是只散落在脚本头部

例如：

- 掌机 Windows 的 watchdog / SSD / 桌面快捷入口 → 写到 `掌机（Windows）` 文档
- 公司 Linux 的 systemd / resume-watch / Linux 维护项 → 写到 `公司（Linux）` 文档

### 如果修改影响所有机器

必须标注：

- `适用机器：通用`
- `系统 / OS：通用`

并在需要时补充说明“哪些部分会在 Windows 生效、哪些部分会在 Linux 生效”。

## 第五步：公司（Linux）环境下的默认读取规则

当当前机器被识别为 `公司（Linux）` 时：

### 默认优先读取

1. `HOST_CONTEXT.md`
2. `docs/多机器-读取与更新规则.md`
3. `docs/公司-Linux-OpenClaw-维护说明.md`
4. 相关 Linux 脚本与配置文件，例如：
   - `scripts/openclaw-resume-watch.sh`
   - `~/.config/systemd/user/openclaw-resume-watch.service`
   - `~/.config/systemd/user/openclaw-resume-watch.timer`

### 默认优先更新

- 公司 Linux 专用维护说明
- Linux 相关脚本
- Linux 相关 systemd 单元说明
- 通用规则（仅当确实影响所有机器时）

### 默认不要误动

- 掌机（Windows）专用 `.ps1` / `.cmd`
- Windows 计划任务说明
- 桌面快捷方式相关内容

除非任务本身明确要求跨机器统一调整。

## 第六步：掌机（Windows）环境下的默认读取规则

当当前机器被识别为 `掌机（Windows）` 时：

### 默认优先读取

1. `HOST_CONTEXT.md`
2. `docs/多机器-读取与更新规则.md`
3. `docs/掌机-Windows-OpenClaw-维护说明.md`
4. 相关 Windows 脚本与本地说明，例如：
   - `scripts/openclaw-gateway-watchdog.ps1`
   - `scripts/stop-openclaw-gateway-zhangji-windows.ps1`
   - `scripts/start-openclaw-gateway-zhangji-windows.ps1`
   - `scripts/optimize-ssd-trim-zhangji-windows.ps1`

### 默认优先更新

- 掌机 Windows 专用维护说明
- Windows 相关脚本
- 计划任务 / 桌面入口说明
- 通用规则（仅当确实影响所有机器时）

### 默认不要误动

- 公司（Linux）专用 shell / systemd 配置

## 第七步：同步更新的统一理解

当用户说：

- `同步这台机器`
- `更新这台机器`
- `拉一下最新规则`

默认理解为：

1. 在当前机器的 workspace 仓库执行 `git pull --ff-only`
2. 如果仓库有变化，或用户明确要求立即生效，再执行 gateway 重启

说明：

- 在 Windows / 掌机上，按本机 gateway 方式处理
- 在 Linux / 公司机上，按本机 gateway / service 方式处理

## 第八步：最终判断原则

一句话规则：

> 先识别当前机器，再读“当前机器匹配”的说明，最后才读通用说明；其他机器的文档只能参考，不能直接执行。
