## 一、问题描述

### 1.1 错误信息

```
[救命 1 错误] 主 session 文件不存在或无法解析: 
sid= file=/home/missyouangeled/.openclaw/agents/main/sessions/.jsonl
```

### 1.2 触发频率

救命 1 cron 配置: 每 5 分钟跑一次 (`*/5 * * * *`)
失败历史: 16:05 / 16:35 / 17:05 连续 3 次都 error
(我清理会话动作发生在 17:40,远晚于首次失败时间)

### 1.3 根因

`scripts/emergency1-orchestrator.py` 第 56 行的 `load_main_session_meta()`:

```python
main = (data.get('agent:main:main') or {}) if isinstance(data, dict) else {}
sid = main.get('sessionId', '')
session_file = main.get('sessionFile', '') or str(SESSIONS_DIR / f'{sid}.jsonl')
```

期望 `sessions.json` 里存在 `agent:main:main` 这个 key。

**实际情况**:
- `agent:main:main` 是 2026-06 之前的"主会话"key(见多份历史文档)
- 2026-06 后 OpenClaw 改用 dashboard 模式,主会话变为 `agent:main:dashboard:*`
- 但 emergency1 没同步更新,继续找 `agent:main:main`
- sessions.json 里这个 key **从来就不存在**(清理前后都没有)
- 所以 emergency1 **从 16:05 起一直报错**(不是清理造成的)

## 二、修复方案

### 2.1 设计原则

- **不动 dashboard 模式**: dashboard 会话是当前实际的主会话,保留它的语义
- **保留向后兼容**: 如果未来 `agent:main:main` 再次被使用,优先用历史 key
- **fallback 优先级**: agent:main:main > 最新 dashboard > 失败

### 2.2 代码改动

**文件**: `scripts/emergency1-orchestrator.py`
**改动位置**: `load_main_session_meta()` 函数(第 50-60 行)

**改动前**:
```python
def load_main_session_meta() -> dict[str, Any]:
    p = SESSIONS_DIR / 'sessions.json'
    if not p.exists():
        return {'main_sid': '', 'main_file': ''}
    try:

```

**改动后**:
```python
def load_main_session_meta() -> dict[str, Any]:
    """读取主会话元信息(支持 agent:main:main + dashboard fallback)。

    优先级:
      1. agent:main:main (历史 key,如果存在)
      2. fallback: 按 updatedAt 排序的最近 dashboard 会话
      3. 两者都找不到 → 返回空 + main_source='empty'
    """
    p = SESSIONS_DIR / 'sessions.json'
    if not p.exists():
        return {'main_sid': '', 'main_file': '', 'main_source': 'empty'}
    try:
        data = json.loads(p.read_text())
    except Exception:
        return {'main_sid': '', 'main_file': '', 'main_source': 'empty'}

    if not isinstance(data, dict):
        return {'main_sid': '', 'main_file': '', 'main_source': 'empty'}

    # 1. 首选:agent:main:main (历史 key)
    main = data.get('agent:main:main') or {}
    sid = main.get('sessionId', '')
    session_file = main.get('sessionFile', '') or ''
    if sid and session_file and Path(session_file).exists():
        return {'main_sid': sid, 'main_file': session_file, 'main_source': 'agent:main:main'}

    # 2. Fallback:最近的 dashboard 会话
    dashboard_keys = [
        k for k in data
        if k.startswith('agent:main:dashboard:') and isinstance(data[k], dict)
    ]
    if dashboard_keys:
        try:
            latest_key = max(
                dashboard_keys,
                key=lambda k: data[k].get('updatedAt', 0) or 0,
            )
            latest = data[latest_key]
            latest_sid = latest.get('sessionId', '')
            latest_file = latest.get('sessionFile', '') or str(SESSIONS_DIR / f'{latest_sid}.jsonl')
            if latest_sid and Path(latest_file).exists():
                return {
                    'main_sid': latest_sid,
                    'main_file': latest_file,
                    'main_source': 'dashboard',
                }
        except (ValueError, TypeError):
            pass

    # 3. 两者都找不到
    return {'main_sid': '', 'main_file': '', 'main_source': 'empty'}
```

**附加改动**: `maybe_alert_main()` 函数
- 增加 `main_source` 字段透传到错误信息
- 当 `source == 'dashboard'` 时,只在 outputs 加一行 `[救命 1 info]` 提示,**不报 error**
