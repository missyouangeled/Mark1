## 〇、从 Mark1 到 贾维斯中枢

v1 方案把服务器定位为「贾维斯的托管机」——迁移现有能力即可。但你要的三个新能力，让服务器的角色完全变了：

```
v1: 贾维斯托管机           v2: 贾维斯中枢
     ┌────────┐                  ┌──────────────────────────┐
     │  贾维斯 │                  │        贾维斯中枢         │
     │  +      │                  │                          │
     │  watcher│                  │  贾维斯核心  (Mark1)      │
     │  + cron │                  │  ─────────────────────   │
     └────────┘                  │  开发工作台  (NEW)        │
    只有贾维斯                    │  自部署服务  (NEW)        │
                                 │  远程连接    (NEW)        │
                                 └──────────────────────────┘
                                 → 你的个人基础设施中枢
```

## 二、三个新增能力详细设计

### 2.1 开发工作台（L3）

#### 目标
在任何设备上（公司电脑、掌机、iPad、手机）打开浏览器就能写代码、跑项目。

#### 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| Web IDE | **code-server** (VS Code 官方 Web 版) | 和本地 VS Code 体验一致、插件生态完整 |
| 运行时隔离 | **Docker Dev Containers** | 每个项目独立环境，不互相污染 |
| 前端开发 | Node.js + Vite/Next.js（已内置） | 你熟悉的技术栈 |
| 小程序开发 | **Taro / uni-app** | 不需要 WeChat DevTools，用标准前端代码编译到微信/支付宝/抖音 |
| 图片处理 | Python Pillow + FFmpeg（已有） | 贾维斯已经能跑 |
| 视频处理 | FFmpeg（已有） | 贾维斯已经能跑 |
| 后端 API | Python FastAPI / Node.js Express | 轻量，够用 |
| App 开发 | PWA / Flutter Web | 编译到 Web，不需要 Xcode/Android Studio |

#### 安装

```bash
# 一键安装 code-server
curl -fsSL https://code-server.dev/install.sh | sh

# 作为 systemd 服务运行
systemctl --user enable --now code-server
# 绑定 127.0.0.1:8080，由 Caddy 反向代理到 code.yourdomain.com
```

#### 与贾维斯的关系

```
用户在 code-server 里写代码
   → 代码存在 /srv/projects/（贾维斯能直接读写）
   → 贾维斯 exec 工具可以直接跑编译/测试/部署命令
   → 贾维斯可以帮你管理 git、安装依赖、排查错误
   → 相当于贾维斯是你的「结对编程 buddy」
```

#### 贾维斯能帮你做什么

- 「帮我在 /srv/projects/web/ 创建 Next.js 项目骨架」
- 「看看这个报错是什么原因」
- 「部署到 Vercel / Cloudflare Pages」
- 「批量压缩 /srv/projects/images/ 下的图片」

### 2.2 自部署服务（L4）

#### 设计原则

```
┌─────────────────────────────────────────────────┐
│          Docker Compose 统一管理                    │
│                                                  │
│  每个服务 = 一个 docker-compose.yml 文件            │
│  服务之间用 Docker network 互联                     │
│  数据卷挂载到 /mnt/data/services/<服务名>/          │
│  通过 Caddy 反向代理暴露 Web 界面                    │
│  Portainer 提供可视化面板                           │
└─────────────────────────────────────────────────┘
```

#### 推荐服务举例

| 类别 | 服务 | 用途 | 成熟度 |
|------|------|------|--------|
| 💬 微信 | wechatferry / ComWeChatRobot | 微信机器人框架，登录后可多端共享消息 | ⭐⭐⭐ |
| 💬 微信 | WeChatMsg | 微信聊天记录管理 + 导出 | ⭐⭐⭐⭐ |
| 📁 网盘 | Nextcloud | 私人云盘，文件同步，日历，联系人 | ⭐⭐⭐⭐⭐ |
| 📁 同步 | Syncthing | P2P 文件同步，所有设备自动同步 | ⭐⭐⭐⭐⭐ |
| 📊 监控 | Uptime Kuma | 服务在线监控 + 告警 | ⭐⭐⭐⭐ |
| 🖼️ 图床 | Chevereto / Lsky Pro | 自建图床，Markdown 直接贴 | ⭐⭐⭐⭐ |
| 📝 笔记 | Outline / Trilium | 知识库 + 文档管理 | ⭐⭐⭐⭐ |
| 🔐 密码 | Vaultwarden | Bitwarden 自部署，密码管理 | ⭐⭐⭐⭐⭐ |
| 📋 看板 | Plane / Focalboard | 项目管理看板 | ⭐⭐⭐ |

#### 微信桥接专项说明

你提到的「登录微信后所有设备共享数据」——目前开源方案有几条路线：

| 方案 | 原理 | 状态 | 风险 |
|------|------|------|------|
| **WeChatFerry** | Hook Windows 微信客户端 | 活跃，社区维护 | 需要 Windows 宿主机或 Wine |
| **wechaty + Puppet** | 多种协议适配（Web/Pad/Windows） | 最成熟，生态丰富 | Pad 协议需付费 token |
| **ComWeChatRobot** | 注入微信进程 | 活跃 | 仅 Windows |
| **WeChatMsg** | 解析本地数据库 | 纯读数据，不交互 | 最安全，只读不写 |

**建议路线**：
1. 先部署 **WeChatMsg**（只读数据库，完全安全）→ 实现聊天记录备份和多端查阅
2. 如果服务器是 Linux（没有 Windows），微信客户端本身跑不了 → 考虑在服务器上开一个 Windows 虚拟机（Docker wine）或远端 Windows 机器上跑 WeChatFerry，再通过 Tailscale 把消息桥接回服务器
3. 或者直接用 **wechaty + 官方 Web 协议**（功能有限但够用）

> ⚠️ 微信对自动化有严格风控，任何非官方客户端都有封号风险。建议先用只读方案（WeChatMsg）试水，确认没问题再上交互方案。

#### 安装模板

```bash
# Docker + Docker Compose 一键安装
curl -fsSL https://get.docker.com | bash
sudo usermod -aG docker jarvis

# Portainer（容器管理 UI）
docker volume create portainer_data
docker run -d \
  -p 127.0.0.1:9443:9443 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  --name portainer \
  --restart unless-stopped \
  portainer/portainer-ce

# 服务目录结构
mkdir -p /srv/docker/{wechat,nextcloud,syncthing,uptime,chevereto}
# 每个服务一个 docker-compose.yml
```

#### 与贾维斯的关系

```
Portainer UI（人工管理）  ←→  贾维斯（自动管理）
   │                              │
   ├─ 手动启动/停止容器            ├─ exec 'docker ps' 查看状态
   ├─ 查看容器日志                 ├─ exec 'docker compose up -d'
   └─ 手动修改配置                 └─ exec 'docker compose logs --tail=50'
                                  └─ 服务挂了自动告警（通过 watcher）
```

### 2.3 远程连接中枢（L5）

#### 设计目标

```
任何一台你的设备 → 装 Tailscale → 自动和服务器组网 → 服务器成为跳板机
```

#### Tailscale 网状拓扑

```
                     ☁️ Tailscale Coordination Server
                              │
              ┌───────────────┼───────────────────┐
              │               │                   │
         🖥️ 中枢服务器    🖥️ 公司PC         🎮 掌机(Windows)
         100.x.x.1      100.x.x.2         100.x.x.3
              │
              ├── 🖥️ GPU 服务器 (SeetaCloud)
              │    100.x.x.4
              │
              ├── 📱 手机 (Android/iOS)
              │    100.x.x.5
              │
              └── 🏠 家里 NAS / 树莓派
                   100.x.x.6
```

**关键特性**：
- 每个设备自动获得 100.x.x.x 内网 IP
- 端到端 WireGuard 加密
- 不需要公网 IP，不需要开端口
- 免费版支持最多 3 个用户、100 台设备

#### 远程场景支持

| 场景 | 实现方式 |
|------|----------|
| **远程修电脑（公司PC）** | `ssh 公司PC`（经 Tailscale） + `xrdp` 远程桌面 |
| **远程连掌机（Windows）** | `ssh 掌机` 或 RDP（经 Tailscale 隧道） |
| **临时 GPU 卸载** | `ssh GPU服务器` → 跑模型 → scp 结果回服务器 |
| **大型工程分流** | 贾维斯检测任务量 → 自动 `sessions_spawn` 到远端 GPU |
| **文件传输** | `scp` / `rsync` 经 Tailscale，无需 U 盘 |
| **设备管理** | 贾维斯通过 SSH 巡检所有设备健康状态 |

#### SSH 统一配置

```bash
# ~/.ssh/config（服务器上的统一配置）
Host 公司pc
    HostName 100.x.x.2
    User BIT-015
    IdentityFile ~/.ssh/id_ed25519

Host 掌机
    HostName 100.x.x.3
    User 点点
    IdentityFile ~/.ssh/id_ed25519

Host gpu
    HostName 100.x.x.4
    User root
    Port 18786
    IdentityFile ~/.ssh/id_ed25519

Host nas
    HostName 100.x.x.6
    User admin
    IdentityFile ~/.ssh/id_ed25519
```

之后贾维斯可以直接：
```
exec: ssh 公司pc dir C:\Users\BIT-015\Desktop
exec: rsync -avz 掌机:/mnt/data/unity-project/ /srv/projects/unity/
exec: ssh gpu nvidia-smi
```

#### 安装

```bash
# Tailscale 一键安装
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

# 在所有设备上重复此操作
# 手机上从应用商店下载 Tailscale 客户端
```

#### 与贾维斯的关系

```
贾维斯作为中枢的 SSH 客户端
   → 通过 exec 工具 SSH 到任何 Tailscale 设备
   → 执行命令、传文件、巡检健康
   → 大型计算自动卸载到 GPU 节点
   → 帮你整理各台电脑的文件
```

## 三、服务器配置升级

新增三层能力后，配置需要上调：

| 组件 | Mark1 原方案 | 中枢 v2 方案 | 说明 |
|------|-------------|-------------|------|
| CPU | 4 核 | **8 核** | Docker + code-server + 多个服务 |
| 内存 | 8-16 GB | **32 GB** | Docker 容器叠起来吃内存 |
| 系统盘 | 80 GB | **100 GB SSD** | 系统 + Docker 镜像 |
| 数据盘 | 100 GB | **200 GB SSD** | 服务数据 + 开发项目 + 模型缓存 |
| 带宽 | 5 Mbps | **10 Mbps+** | code-server 远程桌面流量 |

**预估月费：¥400-800/月**（8核32G 轻量服务器 / 或入门级 ECS）

> 💡 也可以先 4核16G 起步，不够再升——云服务器升级配置很简单。

```
┌────────────────────────────────────────────────────────────┐
│                   贾维斯能力矩阵（中枢版）                    │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  🧠 核心（Mark1）                                          │
│  ├─ 日常陪伴聊天                                           │
│  ├─ 工作协作（代码/文档/调研）                               │
```
