### 2.1 网络方案二选一

| 方案 | 需要 | 优点 | 缺点 | 月费 |
|------|------|------|------|------|
| **A: 公网 IP + 域名** | 服务器有公网 IPv4 + 域名 + DNS | 直连低延迟，完全自主 | 需备案（国内），需 SSL 维护 | IP ~¥30-50/月，域名 ~¥50/年 |
| **B: Cloudflare Tunnel** | 域名 + Cloudflare 账号 | 无需公网 IP / 自带 SSL / 防 DDoS / 不用备案 | 延迟略高（经 CF 中转） | 免费 |

**推荐方案 B**（Cloudflare Tunnel）：成本低、安全、省心。后续可按需升级到方案 A。

## 四、迁移步骤（12 步）

### 步骤 1: 服务器基础环境

```bash
# 1.1 创建用户（非 root 运行）
adduser jarvis
usermod -aG sudo jarvis

# 1.2 更新系统
sudo apt update && sudo apt upgrade -y

# 1.3 安装基础依赖
sudo apt install -y \
  git curl wget build-essential \
  python3 python3-dev python3-venv python3-pip \
  ffmpeg libsndfile1 espeak-ng \
  ca-certificates gnupg

# 1.4 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# 1.5 安装 Caddy（反向代理 + SSL）
sudo apt install -y debian-keyring debian-archive-keyring
curl -1sLf 'https://dl.cloudfoundry.org/debian/caddy-stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/cloudfoundry-caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudfoundry.org/debian/caddy-stable/debian/caddy-stable.list' | sudo tee /etc/apt/sources.list.d/cloudfoundry-caddy-stable.list
sudo apt update && sudo apt install -y caddy

# 1.6 创建数据盘挂载点
sudo mkdir -p /mnt/data/openclaw
sudo chown jarvis:jarvis /mnt/data/openclaw

# 1.7 安装 uv (Python 包管理器)
curl -LsSf https://astral.sh/uv/install.sh | sh
# 添加到 PATH：export PATH="$HOME/.local/bin:$PATH"
```

### 步骤 8: 网络与访问配置

#### 方案 B: Cloudflare Tunnel (推荐)

```bash
# 8.1 安装 cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 \
  -o /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared

# 8.2 登录 Cloudflare
cloudflared tunnel login
# 浏览器打开链接，授权你的域名

# 8.3 创建隧道
cloudflared tunnel create jarvis-tunnel

# 8.4 配置 DNS
cloudflared tunnel route dns jarvis-tunnel jarvis.yourdomain.com
# 替换 yourdomain.com 为你的实际域名

# 8.5 配置并运行隧道
cat > ~/.cloudflared/config.yml << 'EOF'
tunnel: <tunnel-id>
credentials-file: ~/.cloudflared/<tunnel-id>.json

ingress:
  - hostname: jarvis.yourdomain.com
    service: http://localhost:18788
  - service: http_status:404
EOF

# 8.6 安装为 systemd 服务
sudo cloudflared service install

# 8.7 验证
curl https://jarvis.yourdomain.com/
# 应该返回 Control UI 页面
```

#### 可选：Caddy + 公网 IP (方案 A)

```bash
# 编辑 /etc/caddy/Caddyfile
cat > /etc/caddy/Caddyfile << 'EOF'
jarvis.yourdomain.com {
    reverse_proxy localhost:18788
    encode gzip
    header {
        X-Forwarded-Proto https
        X-Real-IP {remote_host}
    }
}
EOF

sudo systemctl restart caddy
```

### 步骤 11: 切换与回滚

#### 11.1 灰度切换

```
阶段 1: 新服务器并行运行 1-3 天
        - 两个实例同时在线
        - 主要在旧机使用，新机仅测试

阶段 2: 主用新服务器
        - 所有日常聊天走新服务器
        - 旧机保持在线但不主动使用

阶段 3: 旧机关闭
        - 确认新服务器稳定 1 周后
        - 旧机 Gateway 可关闭
        - 旧机 workspace 不再更新
```

#### 11.2 回滚预案

如果新服务器出问题：

1. **立即回切旧机**：旧机 Gateway 重新启动即可，workspace 还在
2. **数据不丢**：workspace 由 Git 同步，两边都有
3. **会话不丢**：如果启用了 Git 同步，新服务器的聊天记录可以合并回去

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| Python 依赖版本漂移 | 中 | 高 | 锁定版本号，记录在 requirements.txt；保留旧机 venv 快照 |

| GPU 不可用（轻量服务器无 GPU） | 高 | 中 | CPU 推理可用（ChatTTS 纯 CPU 已验证 OK）；后续可加 GPU 实例 |
| session 文件迁移丢失 | 低 | 高 | 迁移前全量备份、Git push 所有改动 |
