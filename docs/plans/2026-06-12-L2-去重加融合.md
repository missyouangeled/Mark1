# L2 层优化方案：去重 + BM25/embedding 双通道加权融合

**日期**：2026-06-12

## 问题

1. **去重**：同一段话出现在两个文件里（如 transcript + stories 同步），embedding 返回两条相同内容不同位置的结果
2. **孤立路由**：L2 和 L2.5 各自独立短路，不能互相补充。BM25 擅长精确关键词，embedding 擅长语义理解——但现在是「要么用这个、要么用那个」，而不是「两个结果取最好的」

## 方案

### 问题 1：去重（低风险，直接改）

在 `search_l25_embed` 返回结果前，按 content hash 去重，保留 score 最高的那条 → 直接在 `embed-sidecar.py` 的 `do_search` 函数里加。

### 问题 2：加权融合（小重构）

**当前**：
```
L1 → LS → L2(BM25 only) → L2.5(embed only) → L3 → L4
```

**改后**：
```
L1 → LS → L2(融合: BM25 + embedding) → L3 → L4
```

融合策略：RRF（Reciprocal Rank Fusion）——简单、无需调参、跨通道可比。

```
RRF_score(doc) = Σ 1/(k + rank_in_channel)
k = 60（标准值）
```

- L2 层算两个通道：BM25（已有）+ embedding（已有 L2.5 逻辑）
- 两个通道各自跑 top-5，用 RRF 合并排序
- 置信度取 max(BM25_confidence, embedding_confidence)
- 短路条件：融合后的 top_score ≥ 0.7

**删掉 L2.5 独立层，并入 L2。**

### 影响范围

| 文件 | 改动 |
|------|------|
| `scripts/embed-sidecar.py` | `do_search` 加 content hash 去重 |
| `scripts/memory-search-router.py` | 删除 L2.5 层 → 把 embedding 并入 L2 融合函数 |
