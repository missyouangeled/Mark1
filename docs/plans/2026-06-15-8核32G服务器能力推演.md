# 8核32G 服务器能力推演

> 基于当前 Mark1 实际运行数据推算
> 当前环境：4核 8GB VMware 虚拟机

---

## 当前资源消耗（实测）

| 进程 | 内存 | 说明 |
|------|------|------|
| embed-sidecar（向量模型） | **1.3 GB** | 模型常驻内存，吃内存大户 |
| Gateway（Node.js） | **416 MB** | 会话管理+模型路由 |
| infos-handle + Caddy | ~70 MB | 健康数据 API |
| **Mark1 核心合计** | **~1.8 GB** | — |
| GUI（Firefox + GNOME） | ~1.5 GB | 服务器无 GUI，省掉 |

> 当前机器 4核 8GB，跑 Mark1 + 桌面浏览器，还剩 3.5GB 可用。内存不是瓶颈，CPU 偶尔吃紧（embed 推理时）。

## 8核32G 能跑什么——逐层推算

```
总内存:                          32 GB
─────────────────────────────────────────
系统底子 (Ubuntu Server 无 GUI)    -0.5 GB
─────────────────────────────────────────
L2 贾维斯核心 (Mark1)              -2.0 GB    Gateway + embed + infos-handle
─────────────────────────────────────────
L3 开发工作台 (code-server)        -0.5 GB    VS Code Web 版
─────────────────────────────────────────
L4 自部署服务 (Docker 全家)
  Portainer                         -0.1 GB
  Nextcloud + MySQL                 -0.8 GB    私人网盘
  Syncthing                         -0.2 GB    文件同步
  Uptime Kuma                       -0.1 GB    监控面板
  Lsky Pro (图床)                   -0.1 GB
  WeChat 查阅器 (nginx)             -0.1 GB
  预留 3 个未来服务                  -1.5 GB    每个 0.5GB
─────────────────────────────────────────
L5 远程连接
  Tailscale                         -0.1 GB    几乎无感
─────────────────────────────────────────
TTS 语音推理 (按需加载)             -4.0 GB    ChatTTS/XTTS，跑完释放
─────────────────────────────────────────
磁盘缓存 (Linux 自动)              -22.0 GB    读写加速
─────────────────────────────────────────
剩余可用                           ~22 GB     灵活分配
```

## 实际能做什么——场景推演

### 日常运行（所有服务同时在线）

```
✅ 贾维斯 全天候在线（Control UI / WebChat）
✅ 6 个 watcher 守护全部跑着
✅ 5 个 cron 自动化任务
✅ code-server 开着等你随时写代码
✅ Nextcloud 网盘同步文件
✅ Syncthing 多设备文件同步
✅ 图床服务（贴图直接上传）
✅ Uptime Kuma 监控所有服务
✅ Portainer 容器管理面板
✅ Tailscale 所有设备组网

→ 内存占用约 6-7 GB，剩 25 GB 富裕
```

### 开发场景（用 code-server 写项目）

```
✅ 跑一个 Next.js 开发服务器（~300MB）
✅ 跑一个 Python FastAPI 后端（~200MB）
✅ 跑一个 PostgreSQL 本地测试库（~200MB）
✅ FFmpeg 视频转码（吃 CPU，内存 ~1GB）
✅ 同时开着贾维斯聊天

→ 额外占用约 2-3 GB，剩 22+ GB
```

### 峰值场景（TTS 语音 + 开发 + 全部服务）

```
✅ ChatTTS 推理一段 30 秒语音（峰值 ~4GB，跑完释放）
✅ Nextcloud 正在同步大文件
✅ 你正在 code-server 里写代码
✅ Docker 7 个容器全部在线

→ 峰值约 12 GB，剩 20 GB
→ 8 核 CPU: embed 推理用 1-2 核，TTS 用 2-4 核，还有余量
```

## 能力边界——这个配置能做什么/不能做什么

### ✅ 能做

| 场景 | 细节 |
|------|------|
| 网页开发 | Next.js / Vue / React，Vite 热更新，同时跑 3-5 个 dev server |
| 小程序开发 | Taro / uni-app，编译到微信/支付宝/抖音 |
| API 后端 | FastAPI / Express，撑几百并发没问题 |
| 图片处理 | 批量压缩、格式转换、Pillow 脚本 |
| 视频处理 | FFmpeg 转码、剪辑、抽帧（吃 CPU 但 8 核够用） |
| 数据库 | PostgreSQL + MySQL + Redis 同时跑 |
| 私人网盘 | Nextcloud，5-10 个设备同步 |
| 文件同步 | Syncthing P2P，大文件没问题 |
| 图床 | Lsky Pro / Chevereto，日几千张 |
| 远程开发 | code-server（最多同时 2-3 人） |
| AI 语音 | ChatTTS / XTTS 本地推理（纯 CPU，短句 5-30 秒） |
| 向量搜索 | embed-sidecar 全天候在线 |
| 微信记录 | WeChatMsg Web 查阅 |
| 远程管理 | SSH 到所有 Tailscale 设备 |

### ⚠️ 能跑但吃力

| 场景 | 限制 |
|------|------|
| 大模型本地推理 | 7B 参数模型刚好能跑（~14GB 显存/内存），13B 以上吃力 |
| 语音克隆 | XTTS 微调可以（CPU 慢），实时流式不行 |
| AI 绘画 | Stable Diffusion CPU 推理 ~5-10 分钟一张，建议加 GPU |
| 视频直播转码 | 单路 1080p 可以，多路吃力 |
| 大型编译 | Rust/Go 大型项目编译 8 核够用，但别同时跑多个重编译 |

### ❌ 做不了或很不划算

| 场景 | 原因 |
|------|------|
| 微信原生登录 | 无 Linux 客户端（走桥接方案） |
| 大型 AI 训练 | 无 GPU，32GB 不够存梯度 |
| 游戏服务器 | 高并发低延迟需要更高主频+专线 |
| 视频渲染农场 | 需要 GPU |
| 上百人同时访问 | 8 核会瓶颈（但个人/小团队够） |

---

## 一句话总结

> 8核32G 对你目前和可预见的未来需求来说，**绰绰有余**。
> Mark1 全家桶 + 网盘 + 图床 + 文件同步 + 微信查阅 + Web VS Code + 全部 Docker 服务 = 日常 6-7GB。
> 剩下 25GB 用来跑 TTS 推理、磁盘缓存、未来加新服务，空间很大。
> 唯一天花板是**没有 GPU**——如果你以后想做 AI 绘画或本地跑大模型，需要额外租 GPU 实例通过 Tailscale 接入。

---

## 要不要降？

如果预算紧张，**4核16G 也不是不能跑**——Mark1 现在就在 4核8G 上跑着。加上 Docker 全家桶，16G 刚好够（日常 10-12GB，TTS 峰值走 swap）。

但考虑到你要长期用、加新服务、偶尔跑重活，32GB 买的是「不用操心内存」的省心。
