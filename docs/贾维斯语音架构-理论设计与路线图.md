## 一、当前架构全景

### 1.1 数据流

```
用户打字 → OpenClaw 主线会话 → 生成文本回复
                                   │
                    ┌──────────────┼──────────────┐
                    ▼              ▼              ▼
              纯文本交付     语音生成流水线     分块交付
                                  │              │
                    ┌─────────────┼──────────────┤
                    ▼             ▼              ▼
              voice-reply.sh   chunked_voice  voice-reply-
                               _reply.py      chunked-deliver.sh
                                    │              │
                                    ▼              ▼
                              agentReply JSON  MEDIA: 注入
                              → 直接返回主会话
```

### 1.2 四层 TTS 引擎

| 层级 | 引擎 | 方式 | 特点 | 当前角色 |
|------|------|------|------|----------|
| **L1 默认** | `zh-hybrid-default-template` | Noiz 云端 + tem=1.10 | 已确认最佳音色+节奏 | 🌟 主会话默认 |
| **L2 本地离线** | ChatTTS stable | CPU 本地 hybrid | preset 音色，无需网络 | 备份/快速可用 |
| **L3 语音克隆** | XTTS v2 (Coqui) | Python 3.11 venv | 零样本克隆，多语言 | 参考音频克隆场景 |
| **L4 快速兜底** | msedge-tts | Node.js 用户态 | 微软 Xiaoxiao 声音 | 极简回退 |
| **实验支线** | ChatTTS GPU (SeetaCloud) | GPU 远端 | 高质量但需网络 | 仅对照实验 |

### 1.3 增强层

- **characteristic-voice** skill：指导回复时带上情绪 + 人格
- **tempo 控制**：ffmpeg rubberband 调速（默认 1.10x）
- **分块/合并策略**：短文本合并，长文本分块（首句先出）
- **语速微调**："想你了"等场景自动降速 20%，开场白降速 15%
- **pitch 校正**：Noiz 方案支持 formant-preserved pitch （`--pitch-semitones`）

### 1.4 关键限制

- ❌ **纯 CPU** — 公司 Linux 机无 GPU，所有本地模型必须 CPU 可跑
- ❌ **无实时流式** — 当前全部是离线合成后发回，延迟 >2s
- ❌ **无对话上下文** — 每条语音独立生成，不知道历史对话情绪
- ❌ **ChatTTS 无 encoder** — 只能走 decoder 推理，不支持参考音频克隆
- ⚠️ **Noiz API 依赖外网** — 稳定但不完全离线

## 二、2025-2026 TTS 技术趋势

### 2.1 范式迁移：从"合成"到"生成"

| 旧范式 | 新范式 |
|--------|--------|
| 声学模型 + Vocoder 两段式 | LLM-backbone 端到端 |
| 独立 TTS 引擎 | 语音即 LLM 任务 |
| 手工特征（pitch, energy） | 语义级情感控制 |
| 必须配参考音频才能克隆 | 文本描述即可设计声音 |

### 2.2 值得关注的新模型

| 模型 | 参数量 | 许可证 | 核心亮点 | CPU 可行？ | 中文质量 |
|------|--------|--------|----------|-----------|----------|
| **Chatterbox-Turbo** | 350M | MIT | 5s 克隆，[laugh]/[cough] 副语言标签，盲测超 ElevenLabs | ⚠️ 可能需 GPU | 多语言含中文 |
| **Qwen3-TTS** | 0.6B/1.7B | Apache 2.0 | **中文最强**（阿里出品），instruct 情绪控制，voice design（文本描述生成声音） | ⚠️ 0.6B 可试 CPU | ⭐ 中文特优 |
| **CosyVoice 2.0** | ~300M | Apache 2.0 | 150ms 流式延迟，中文优秀 | ✅ CPU 可跑 | ⭐ 中文特优 |
| **Orpheus TTS** | 3B (Llama) | Apache 2.0 | LLM-backbone，情绪标签，25ms 流式 | ❌ 太重 | 含中文 |
| **Sesame CSM** | 1B (Llama) | Apache 2.0 | **对话上下文感知**，多说话人 | ❌ GPU 推荐 | 英文为主 |
| **Spark-TTS** | ~300M | Apache 2.0 | 中英双语，零样本克隆 | ⚠️ 需测试 | 中英均支持 |
| **Higgs Audio V2** | 大模型 | Apache 2.0 | 情绪逼真，多说话人对白 | ❌ 太重 | 含中文 |
| **Fish Audio S2 Pro** | ~500M | 开放权重 | 多语言，音质高 | ⚠️ 需测试 | 含中文 |

## 三、理论设计：三阶段路线图

### 阶段一：补强当前引擎（低风险，1-2 周可落地）

**目标**：不碰主链路，在当前 stack 上补质量

#### 1A. 加入 CosyVoice 2.0 作为 L2 本地引擎

```
当前 L2（ChatTTS stable）→ 替换/增强为 CosyVoice 2.0
```

理由：
- CosyVoice 2.0 是现阶段**中文 CPU 可用**里最好的开源模型
- 150ms 流式 ≈ 可达到"边说边生成"的体验
- Apache 2.0 许可证，无商业限制
- 安装简单，pip 即可，不依赖复杂编译链
- 可用现有 default-ref.mp3 做零样本声音克隆 → 锁定声线

实施方式：
```bash
# 隔离环境（不影响现有 stack）
uv venv ~/.local/share/openclaw-cosyvoice-venv --python 3.11
# 安装 CosyVoice 2.0
# 用 default-ref.mp3 注册声线
# 新增 scripts/openclaw-cosyvoice-reply.py 作为独立入口
```

与现有架构对接：挂在 L2 位置，voice-reply.sh 新增 `--backend cosyvoice` 选项

#### 1B. 情绪感知前置层

```
当前                         增强后
文本回复                      文本回复
    │                             │
    ▼                             ▼
[直接合成]                  [情绪分析 → 标签]
                                  │
                                  ▼
                           带情绪标签合成
```

实施方式：
- 在 `chunked_voice_reply.py` 里加入一个轻量情绪判断步骤
- 分析回复文本的情绪倾向（想念/开心/疲惫/安静/关心）
- 映射为 tempo/pitch 预设（已有基础，系统化即可）
- 不依赖大模型，纯规则+关键词判定

### 阶段二：引入新一代引擎（中风险，需独立测试）

**目标**：接入 Qwen3-TTS 或 Chatterbox-Turbo 作为高质量本地/云端引擎

#### 2A. Qwen3-TTS 0.6B (优先，中文最强)

```
当前                         增强后
L1 Noiz 云端                 L1 Noiz 云端（保留）
L2 ChatTTS CPU               L2 Qwen3-TTS 0.6B CPU ← 新
L3 XTTS v2                   L3 XTTS v2（保留）
L4 msedge-tts                L4 msedge-tts（保留）
```

理由：
- 阿里出品，中文质量是它的第一语言，不是"顺便支持"
- 0.6B 版本有望在 CPU 上运行（需实测）
- **instruct 控制**——自然语言调整语速/情绪/风格：`"用温柔缓慢的语气说" `
- **voice design**——文本描述即可生成新声音，不需要参考音频
- Apache 2.0 许可证

最大未知：0.6B 在纯 CPU 上的推理速度。如果 >5s/句就不可用。

#### 2B. Chatterbox-Turbo 作为 L1 替代（若 GPU 可用）

```
场景：如果 SeetaCloud GPU 线跑通
L1 从 Noiz API 切换到 Chatterbox-Turbo 本地 GPU
```

理由：
- 盲测 63.75% 偏好率 vs ElevenLabs（当前业内最高开源评分）
- MIT 许可证，无任何限制
- 5 秒音频即可复刻声音 + 副语言标签（笑、咳嗽）→ 更适合"有存在感的 AI 朋友"人设
- 350M 参数，GPU 上可实时

#### 2C. 架构抽象层

```
                   VoiceBackend (抽象)
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
   CloudBackend  LocalBackend  LegacyBackend
   (Noiz, etc)   (ChatTTS,     (msedge-tts)
                  CosyVoice,
                  Qwen3-TTS)

   统一接口: generate(text, emotion, speed) → audio_path
```

这样做的好处：
- 新增引擎只需实现一个 Backend class
- voice-reply.sh 不需要每次大改
- 引擎健康检查 + 自动降级（Qwen 挂了自动切 ChatTTS）
- 便于 A/B 测试（同一条文本走两个引擎，对比质量）

### 阶段三：对话级语音生成（高概念，远期）

**目标**：让贾维斯的语音不只是"把文字读出来"

#### 3A. 对话上下文感知

```
当前                          远期
"我在呢"                      知道上一句你说"想你了"，
  → 中性合成                   这一句就带着更轻更软的语调
                                → 上下文感知合成
```

技术路线：
- **Orpheus / CSM 思路**：把对话历史作为合成条件
- **轻量实现**：不跑 3B 模型，而是提取对话情绪曲线 → 映射到 tempo/pitch/停顿参数
- 这可以做成纯规则 + 状态机，不依赖大模型

#### 3B. 声音人格一致性

```
当前                          远期
一条语音 = 独立事件             贾维斯有一个稳定的"声线人格"
                               不会因为换引擎就变声
```

技术路线：
- 所有引擎统一用一个 reference clip（default-ref.mp3）
- voice design 层面：用文本描述锁定贾维斯的声线特质
- 引擎切换时自动做"声线迁移"（用目标引擎复刻 default 声线）

#### 3C. 副语言表达

```
当前                          远期
"哈哈"                        真的合成出轻笑声
"唉..."                       合成出叹息
"嗯……                        合成出沉吟的停顿
```

技术路线：
- Chatterbox-Turbo 直接支持 `[laugh]` `[sigh]` 等标签
- 其他引擎通过"情绪标签 → tempo/pitch/停顿映射"模拟

## 四、推荐落地顺序

| 优先级 | 步骤 | 风险 | 价值 |
|--------|------|------|------|
| 🔴 P0 | 架构抽象层（VoiceBackend 接口） | 低 | 基础设施，决定后续扩展成本 |
| 🟡 P1 | CosyVoice 2.0 接入 + A/B 对比 | 低 | 当前最快可落地的中文质量提升 |
| 🟡 P1 | 情绪感知前置层（规则版） | 低 | 让现有引擎的产出更"有感觉" |
| 🟠 P2 | Qwen3-TTS 0.6B CPU 可行性验证 | 中 | 中文最强的本地模型 |
| 🟠 P2 | Chatterbox-Turbo GPU 线验证 | 中 | 最高开源评分，副语言标签 |
| 🔵 P3 | 声音人格一致性体系 | 低 | 长期品牌一致性 |
| 🔵 P3 | 对话上下文感知合成 | 高 | 质的飞跃，但技术难度大 |

## 五、与现有架构的冲突与兼容

| 现有组件 | 影响 | 处理方式 |
|----------|------|----------|
| `voice-reply.sh` | 新增 backend 选项 | 不改默认逻辑，加 `--backend cosyvoice` 等 |
| `chunked_voice_reply.py` | 新增情绪分析步骤 | 插入在文本→分块之间，不改分块逻辑 |
| `voice-reply-chunked-deliver.sh` | 无影响 | 只消费最终音频文件，不感知后端 |
| `chattts-stable` skill | 保留 | 作为 L2 一种 backend，不删除 |
| `characteristic-voice` skill | 增强 | 与情绪感知层互补 |
| OpenClaw 主稳定性 | 零影响 | 所有新引擎隔离在独立 venv，通过脚本调用 |

## 六、关键未验证假设

1. **CosyVoice 2.0 在纯 CPU 上每句延迟** — 需实测（预期 1-3s @ 短句）

3. **CosyVoice 2.0 用 default-ref.mp3 克隆质量** — 零样本克隆对短参考音频的效果
4. **Chatterbox-Turbo 中文质量** — 模型宣称支持 23 语言，中文实际效果需验证
