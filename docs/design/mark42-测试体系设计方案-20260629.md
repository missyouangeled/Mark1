# Mark42 — 测试体系设计方案

> **作者**: 贾维斯 (agent:main:main)
> **日期**: 2026-06-29
> **状态**: 📋 设计方案，待点点评审
> **背景**: 综合体检报告把"测试覆盖 5.0/10"列为最大短板。本方案从零搭建 Mark42 的正式测试体系。

---

## 一、现状盘点（基于实际代码扫描）

### 已有测试
| 文件 | 行数 | 类型 | 范围 |
|---|---|---|---|
| `scripts/mark42_modules/test_session_fence.py` | 7.3KB | 集成测试 | armor 写入 session 的 fence 防护 |
| `scripts/mark42_modules/test_day4_integration.py` | 10.6KB | 集成测试 | armor + algo_scheduler 联动 |
| `scripts/mark42-tests.py` | ~200 行 | **烟测脚本** | 守护进程存活性 + CLI 输出快照 |

### 缺失项
- ❌ **没有 pytest**（系统未装）
- ❌ **没有 conftest.py / fixtures**
- ❌ **没有 unit-level 单测**（模块内部函数）
- ❌ **没有持续集成**（每次改动靠人肉跑 `mark42-tests.py`）
- ❌ **没有覆盖率门禁**（不知道哪些代码没被覆盖）
- ❌ **CI 上跑守护进程** = 真生产环境（烟测脚本会有副作用污染风险）

### 模块特征（影响测试设计）
- **6 个模块用了 subprocess**（armor / cli / engine / session_fence_safe / utils / test_day4_integration）→ 必须 mock 或隔离
- **4 个模块用了 IO/网络**（armor / compress_queue / engine / llm_text_compressor）→ 必须 tmp_path 隔离
- **全局路径硬编码**：所有模块通过 `XDG_STATE_HOME` 环境变量读状态目录 → 可通过 `monkeypatch.setenv` 隔离 ✅
- **所有模块都能正常 import**（已验证）→ 模块本身无循环依赖
- **20 个模块** 中 6 个是大块核心（armor/engine/heavy/cli/utils/config），其余 14 个是压缩/工具子模块

## 二、设计目标

| 目标 | 量化指标 |
|---|---|
| 测试体系完整 | 6 个核心模块每个有 unit 测，跨模块联动有 integration 测 |
| 可在 CI 跑 | 不依赖 systemd / 不写真生产状态 / 全部用 tmp_path 隔离 |
| 跑得快 | 完整套件 < 30 秒（pytest-xdist 并行后 < 10 秒） |
| 覆盖率 | 核心模块 (armor/engine/heavy/cli) **≥ 80%**，整体 ≥ 70% |
| 不破坏现有 | `mark42-tests.py` 烟测脚本保留并接入新体系 |
| 复用 fixture | 一个 conftest.py 提供共享 fixture，所有测试零样板 |

## 三、目录结构（推荐）

```
scripts/
├── mark42.py                      # 主入口（不动）
├── mark42-tests.py                # 现有烟测脚本（保留 → 改为 thin wrapper）
├── mark42_modules/                # 源码（不动）
│   ├── ...
│   ├── test_session_fence.py      # 保留，作为 integration test 入口
│   └── test_day4_integration.py   # 保留，作为 integration test 入口
│
└── tests/                         # 🆕 正式测试目录
    ├── __init__.py                # 空文件（让 pytest 识别为 package）
    ├── conftest.py                # 🔑 共享 fixtures（环境隔离、路径、CLI runner）
    │
    ├── unit/                      # 单元测试（不跨模块）
    │   ├── __init__.py
    │   ├── test_config.py         # config.py：路径解析、阈值常量
    │   ├── test_utils.py          # utils.py：JSON/file/broker helper
    │   ├── test_armor_compress.py # armor.py：核心压缩逻辑（mock subprocess）
    │   ├── test_armor_guard.py    # armor.py：守护进程模式（mock 自身循环）
    │   ├── test_engine_daemon.py  # engine.py：daemon 心跳、loop 注册
    │   ├── test_engine_templates.py # engine.py：context-guard/health-watch/model-fallback
    │   ├── test_heavy.py          # heavy.py：preflight/start/finish/cleanup
    │   ├── test_cli.py            # cli.py：argparse 分发、status 聚合
    │   ├── test_logs.py           # logs.py：日志轮替、截尾
    │   └── test_compression.py    # 压缩子模块（text/code/diff/smart_crusher）
    │
    ├── integration/               # 集成测试（跨模块 + 真 subprocess + 真文件）
    │   ├── __init__.py
    │   ├── test_armor_to_engine.py  # armor 触发 → broker event → engine 响应
    │   ├── test_full_pipeline.py    # 模拟完整：armor.check → engine.tick → broker
    │   ├── test_session_fence.py    # 旧 test_session_fence.py 迁过来
    │   └── test_daemon_lifecycle.py # 守护进程启停（不用 systemd，纯 Python 启动/停止）
    │
    └── smoke/                     # 烟测（需守护，CI 上 opt-in 跑）
        ├── __init__.py
        └── test_live_daemons.py    # 真守护存活检测（仅在本地手跑或 cron 跑）
```

### 为什么这样分
1. **unit/** — 不触发 subprocess、不写真文件，最快、最稳、最多
2. **integration/** — 允许触发 subprocess（用 `pytest-subprocess` fake）或写真文件到 tmp_path
3. **smoke/** — 真正守护进程存活检测，CI 默认不跑，提供 `--runslow` 标记

## 四、conftest.py 设计（核心）

```python
# scripts/tests/conftest.py
"""Mark42 共享 fixtures - 隔离文件系统、配置环境。"""

import os
import sys
import pytest
from pathlib import Path

# 让 import 找到 mark42_modules
TESTS_DIR = Path(__file__).resolve().parent
SCRIPTS_DIR = TESTS_DIR.parent
sys.path.insert(0, str(SCRIPTS_DIR))


# ── 1. 环境隔离：所有测试都用 tmp_path 模拟状态目录 ──

@pytest.fixture(autouse=True)
def isolated_state(monkeypatch, tmp_path):
    """自动应用：每个测试都用临时 XDG_STATE_HOME，零污染。"""
    fake_state = tmp_path / "state"
    fake_state.mkdir()
    monkeypatch.setenv("XDG_STATE_HOME", str(fake_state))
    monkeypatch.setenv("HOME", str(tmp_path / "home"))
    # 强制重新 import config（因为 XDG_STATE 在 import 时就被读取）
    import importlib
    if "mark42_modules.config" in sys.modules:
        importlib.reload(sys.modules["mark42_modules.config"])
    yield fake_state
    # 自动 teardown：tmp_path 由 pytest 清理


# ── 2. 共享 fixture ──

@pytest.fixture
def state_dir(isolated_state):
    """返回状态目录根：~/.local/state/openclaw/mark42"""
    from mark42_modules import config
    return config.MARK42_STATE


@pytest.fixture
def armor_state(state_dir):
    return state_dir / "armor"


@pytest.fixture
def engine_state(state_dir):
    return state_dir / "engine"


@pytest.fixture
def broker_dir(state_dir):
    return state_dir / "broker"


@pytest.fixture
def fake_session_file(tmp_path):
    """模拟一个 OpenClaw session jsonl 文件"""
    session = tmp_path / "sessions" / "agent-main-main.jsonl"
    session.parent.mkdir(parents=True, exist_ok=True)
    session.write_text('{"role":"user","content":"hello"}\n')
    return session


# ── 3. CLI runner（集成测试用）──

@pytest.fixture
def cli_runner():
    """封装 mark42.py CLI 调用"""
    import subprocess
    mark42 = SCRIPTS_DIR / "mark42.py"

    def run(*args, env=None):
        full_env = os.environ.copy()
        if env:
            full_env.update(env)
        return subprocess.run(
            [sys.executable, str(mark42), *args],
            capture_output=True, text=True,
            env=full_env, timeout=30,
        )

    return run


# ── 4. Mock helpers ──

@pytest.fixture
def mock_subprocess(monkeypatch):
    """用 pytest-subprocess 的 fake_process（自动注入）"""
    # 真正的 mock 由 pytest-subprocess 插件提供，无需代码
    # 这里只提供高层 helper
    pass


@pytest.fixture
def sample_messages():
    """标准测试输入：模拟一段对话"""
    return [
        {"role": "user", "content": "你好"},
        {"role": "assistant", "content": "你好，有什么可以帮你的？"},
        {"role": "user", "content": "压缩这段对话"},
        {"role": "assistant", "content": "好的，我来处理。"},
    ] * 20  # 80 条


# ── 5. Marker 注册 ──

def pytest_configure(config):
    """注册自定义 marker"""
    config.addinivalue_line("markers", "slow: 烟测，跑守护进程，默认跳过")
    config.addinivalue_line("markers", "integration: 集成测试，跑 subprocess")


def pytest_collection_modifyitems(config, items):
    """默认跳过 @pytest.mark.slow 测试"""
    if not config.getoption("--runslow", default=False):
        skip_slow = pytest.mark.skip(reason="需要 --runslow 才跑")
        for item in items:
            if "slow" in item.keywords:
                item.add_marker(skip_slow)
```

### 关键设计点
- **`autouse=True`** — 每个测试自动应用环境隔离，**所有测试都零样板**
- **强制 reload config** — 因为 `XDG_STATE` 在 import 时就被读取，必须 reload 才生效
- **`monkeypatch`** — pytest 原生 fixture，自动 teardown，安全
- **cli_runner 封装** — 集成测试用，subprocess + 超时 + 环境隔离一次搞定

## 五、典型测试用例样板

### Unit 测试样板（10 行内能写完一个）
```python
# scripts/tests/unit/test_armor_compress.py
"""armor.py 的压缩逻辑单测。"""
import json
import pytest


def test_armor_compress_writes_to_state(armor_state, monkeypatch):
    """压缩结果应写入 armor_state/actions.jsonl"""
    from mark42_modules.armor import armor_compress

    # mock LLM 调用，避免真网络
    monkeypatch.setattr(
        "mark42_modules.armor._llm_analyze",
        lambda *a, **kw: {"preserved": {}, "discarded": {"summary": "x"}}
    )

    armor_compress(reason="test", content="dummy content")

    actions = armor_state / "actions.jsonl"
    assert actions.exists()
    last = json.loads(actions.read_text().strip().split("\n")[-1])
    assert last["action"] == "compress"
    assert last["reason"] == "test"


def test_armor_check_threshold(state_dir, monkeypatch):
    """阈值检查应在不同使用率下返回不同等级"""
    from mark42_modules import armor
    # mock 使用率
    monkeypatch.setattr(armor, "_current_usage", lambda: 50.0)
    level, _ = armor.armor_check()
    assert level == "ok"

    monkeypatch.setattr(armor, "_current_usage", lambda: 75.0)
    level, _ = armor.armor_check()
    assert level == "warn"

    monkeypatch.setattr(armor, "_current_usage", lambda: 90.0)
    level, _ = armor.armor_check()
    assert level == "alert"
```

## 十、风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| **pytest-subprocess 和现有 subprocess 冲突** | 测试不隔离 | 优先用 monkeypatch + tmp_path，少用 fake_process（只在不得不测 CLI 子进程时用） |
| **reload config 后 import chain 断裂** | 某些模块拿到旧常量 | conftest 里集中处理 reload，被测模块不重复 reload |
| **systemd 守护进程污染** | 测试写真的状态文件 | **完全用 tmp_path**，永不动 `~/.local/state/openclaw/mark42/` |
| **LLM 调用阻塞测试** | CI 超时 | mock `_llm_analyze`，把"真 LLM 路径"放在 smoke 用 `@pytest.mark.slow` |
| **覆盖率门禁太严，初期跑不动** | 阻碍提交 | 阶段 4 之前不加 `--cov-fail-under`，阶段 4 加也只设在 50%，后续按季调到 70% |
| **现有 .pyc 缓存干扰 reload** | 测试 flaky | conftest 启动时 `find . -name __pycache__ -exec rm -rf {} +`（只清测试目录的） |

## 十一、验收标准（Go-Live 条件）

✅ **必达**：
1. `pytest scripts/tests/unit/` 全部通过，6 个核心模块每个 ≥ 10 个 test cases

4. 整体覆盖率 ≥ 60%（阶段 4 完成时）

✅ **可选（Nice to Have）**：

7. 覆盖率 ≥ 80% 后加 `--cov-fail-under=80`
