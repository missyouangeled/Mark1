## 建议项 (建议修复但不阻塞)

### S1. print() 在生产代码中（196 处）

| 文件 | print 数量 |
|---|---:|
| cli.py | 178 |
| installer.py | 22 |

**建议**：CLI 输出用 print 是合理的（CLI 工具本质），但 daemon 类函数应改用 logging。
**优先级**：低。当前不影响功能。

### S2. 打包构建未验证

`python3 -m build` 因环境限制（PEP 668 / venv）未能完成验证。
**需要**：在 CI 环境中验证 `pip install` 和 `python3 -m build` 能正常产出 wheel。
**优先级**：中。CI workflow 里已有覆盖。

### S3. Dockerfile 引用了两个源目录

```dockerfile
COPY --chown=mark42:mark42 mark42-pkg/ ./mark42-pkg/
COPY --chown=mark42:mark42 scripts/mark42_modules/ ./scripts/mark42_modules/
```

**问题**：Docker 镜像同时复制了两套代码，且 `scripts/mark42_modules/` 的依赖未在 Dockerfile 中安装。
**建议**：Dockerfile 只应复制 `mark42-pkg/`，用 `pip install .` 安装。
**优先级**：中。
