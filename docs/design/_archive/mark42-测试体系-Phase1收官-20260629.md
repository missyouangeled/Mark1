## 实战教训（必读）

**写测试**时挖出的真问题：

1. **armor.py `_save_json` 顺序错**：在 `compactTriggered` 字段设置**之前**写文件，导致字段永远丢失 → 修复后 5 个红测试转绿

2. **`from .X import Y` 缓存陷阱**：reload config 后 `heavy.SCRATCH` 仍是旧值 → conftest 必须 reload 后**额外** monkeypatch 依赖模块

3. **status_dashboard 函数体内 import**：mock target 必须用完整路径 `mark42_modules.armor.armor_check`，不能用 `patch.object(cli, ...)`

4. **fcntl 文件锁 + mock**：mock 的 file handle 不能 fileno()，测试要 mock 整个写函数

5. **hard-code 路径不被 XDG_STATE 派生**：`SCRATCH = Path("/mnt/...")` 要 conftest 单独处理

```bash
# 打开 htmlcov/index.html

# 写新测试
# 1. 读 docs/design/mark42-测试手册.md
# 2. 看对应模块的现有 test_<module>.py 样板
# 3. 用 conftest 共享 fixture
# 4. 跑 pytest 验证
```
