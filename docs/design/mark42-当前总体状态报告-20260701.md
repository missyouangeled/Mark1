## 3）本轮顺手补齐的两个工程口径问题

### 问题 A：`memory-embed-search.py` 直接用系统 python 会因缺 `numpy` 失败

原现象：

```bash
python3 scripts/memory-embed-search.py 'Mark42 启动'
```

会报：

- `ModuleNotFoundError: No module named 'numpy'`

#### 已修正

现在该脚本会在系统 python 缺依赖时：

- 自动重执行到：
  - `~/.local/share/openclaw-embed-venv311/bin/python3`

并且：

- `--check` 也已修正为无需强制输入 query

#### 修后验证

以下命令都已通过：

```bash
python3 scripts/memory-embed-search.py --check
python3 scripts/memory-embed-search.py 'Mark42 启动'
```

---

### 问题 B：watchdog 同一条日志会写两次

根因定位：

- 脚本内 `tee -a watchdog.log`
- systemd unit 又将 stdout/stderr append 到同一文件

结果就是：

- 一条日志写两遍

#### 已修正

`tools/mark42-watchdog/mark42-watchdog.sh` 已调整为：

- **systemd 非 TTY 场景**：只输出到 stdout/stderr，由 unit 自己落盘
- **手动终端场景**：继续 `tee -a`，方便现场调试

#### 当前说明

- 历史重复行会保留
- **未来新增日志不应再双写**
