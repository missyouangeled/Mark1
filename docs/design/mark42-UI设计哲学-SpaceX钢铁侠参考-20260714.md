## 一、4 条核心设计原则(SpaceX + 钢铁侠共同提炼)

### 原则 1 · 信息密度极高,但视觉极简

| 维度 | 反例(常见 BI 仪表盘) | 正例(SpaceX 任务控制) |
|---|---|---|
| 一屏参数数量 | 5-10 个 | 30+ 个 |
| 视觉元素 | 图表、动画、装饰 | 纯文字 + 单色块 |
| 配色数量 | 7+ 种 | 3 种(绿/黄/红) |
| 字体大小 | 3-4 级 | 2 级(数字大 / 标签小) |

**Mark42 应用**:
- 现在的 v3 监控应该能在一屏内显示:**8 个核心状态 + 24h 自愈记录 + 当前 L3 学习率 + 待用户确认事项 + 系统负载 + 错误档案命中率** = 30+ 参数
- 全部用纯文字 + 单色块,不用图表(图表占空间大,精度低)

### 原则 2 · 颜色不超过 3 种 + 异常自动"放大"

**SpaceX 配色**(我从 Falcon 9 任务控制截图反复看过):
- 灰:背景 + 正常参数
- 绿:正常 / 成功
- 黄:警告 / 待观察
- 红:危险 / 立即处理

**钢铁侠 HUD 配色**:
- 黑:背景(透明)
- 蓝:数据(默认)
- 琥珀/橙:警告
- 红:危险

**Mark42 应用**:
- Control UI / 战甲 HUD 统一为 **4 色**:灰 / 绿 / 黄 / 红
- **异常自动"放大"**:一切正常时,UI 几乎静态;一出现异常,异常点**主动跳出来**(放大 + 闪烁 + 文字变红)
- 用户的注意力永远被"最重要的事"抓住,不会被"一堆正常参数"淹没

### 原则 3 · 每屏都有"下一步"

**反例**:仪表盘显示"embed 核挂了",然后呢?用户得自己查 log、自己决策、自己操作。

**正例(SpaceX)**:显示"embed 核挂了"+ "自动重启中(预计 30 秒)"+ "若 3 次失败请切换到 L1 关键词模式(点这里)"。**一个屏幕解决"看 → 决策 → 操作"三件事**。

**Mark42 应用**:
- 每个异常显示旁边,必须有 **3 个槽位**:
  1. **现在在干嘛**(自动重启 / 等待 / 已恢复)
  2. **预计多久恢复**(30 秒 / 5 分钟 / 需用户介入)
  3. **你能做什么**(切换备用模式 / 授权修复 / 暂时忽略)

### 原则 4 · 不沉默,但只在值得说时说

**钢铁侠战甲 HUD 的设计哲学**(电影特效师访谈里讲过):
- 数据浮在视野里,不占操作位——信息是"背景",操作是"前景"
- **异常自动语音播报**——不用盯着看,战甲自己说"左臂能量 23%,建议切换备用电源"

**反例**:现在 Mark42 跑了一天什么都没说 → 用户不知道战甲在干嘛 → 觉得"是不是挂了?"

**正例**:Mark42 在以下时机主动说话:
- 早上首次启动 → "早上好,昨晚我修了 3 个问题,详情见 HUD"
- 检测到新类型异常 → "我遇到一个新情况,不确定怎么修,要不要看一下?"
- 自愈完成 → "刚才的 L2.5 索引丢失已自动恢复,耗时 2 分 17 秒"
- 战甲升级完成 → "v3.1 已生效,新能力:主动交流;旧能力:全部保留"
- 重要里程碑 → "今天自愈率 99.2%,首次达到 SRE 业界标准"

**闭嘴的边界**:
- 一切正常时 → 不说话
- 用户没问 → 不主动汇报细节
- 凌晨 0:00-7:00 → 不主动播报(用户可能在睡)

## 二、对比 Mark42 现状(诚实评估)

| 维度 | 现在 | SpaceX / 钢铁侠应该有的 | 差距 |
|---|---|---|---|
| 一屏参数数量 | ~5 | 30+ | **大**(可加) |
| 配色数量 | 5+ | 3-4 | **中**(需收敛) |
| 异常自动放大 | ❌ 无 | ✅ 主动跳出来 | **大**(要做) |
| 看完知道下一步 | ❌ 不知道 | ✅ 知道 | **大**(要加操作) |
| 主动语音/文字汇报 | ❌ 完全静默 | ✅ 异常自动播报 | **大**(要做) |

**当前最痛点(点点 9:01 指出的)**:
> "实际使用的感觉就是能显示问题。但是无法操作,也没有什么后续。"

——这正是"每屏都有下一步" 这条原则没落地。

---

## 三、落地路径(分阶段,不一次做完)

### 阶段 1 · v3.3 内(2-3 周内)

- 配色收敛到 4 色(灰/绿/黄/红)
- 异常自动放大(1 小时工作量,纯前端 CSS)
- 暂时不动"下一步"和"主动汇报"

### 阶段 2 · v4 启动后

- 每屏加"下一步"槽位(3 槽位:状态/预计/操作)
- 战甲主动汇报协议(早上 1 次 + 异常时)
- 语音/文字播报(可关闭,默认开启)

### 阶段 3 · v4 跑稳后

- 信息密度升级到 30+ 参数
- 战甲 HUD 改造(浮在视野里,可选关闭)
