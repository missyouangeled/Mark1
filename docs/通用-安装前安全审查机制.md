# 安装前安全审查机制

> 适用机器：通用
> 系统 / OS：通用
> 建立时间：2026-06-09
> 触发条件：安装任何新工具、脚本、扩展、依赖包之前

---

## 为什么需要

CASE-20260513-001 的教训：一次看似无害的硬件监控工具安装，因为未审查 Ring0 驱动风险，导致 Windows 宿主机崩溃、用户重装系统。从此以后，任何安装都必须先过审查。

---

## 审查清单（四项必过）

### 1. 数据安全

- [ ] 该工具是否会删除/覆盖/移动用户数据？
- [ ] 是否有「全删」「递归删除」「通配删除」入口？
- [ ] 是否操作文件系统、数据库、网络存储？
- [ ] 是否有 dry-run / 确认机制？
- [ ] 出问题后是否有回滚路径？（Git revert / snapshot / backup）
- [ ] 是否对标崩坏案例库中的相似模式？（必读 `docs/对系统操作必须要参考的崩坏案例.md`）

### 2. 系统稳定性

- [ ] 是否需要 root/sudo？
- [ ] 是否涉及内核模块、驱动、Ring0、EC、SMBus、直接 I/O？
- [ ] 是否会修改系统配置（systemd、内核参数、启动项）？
- [ ] 是否会影响已有服务的正常运行？
- [ ] 如果崩溃，最坏后果是什么？恢复成本多大？

### 3. 运行负担

- [ ] CPU 占用预估
- [ ] 内存占用预估
- [ ] 磁盘 I/O 预估
- [ ] 网络带宽/请求量预估
- [ ] 当前机器剩余资源是否够用？

### 4. 依赖审查

- [ ] 需要哪些外部依赖？（系统包、pip、npm、其他）
- [ ] 哪些已安装、哪些需要新增？
- [ ] 新增依赖的安装成本与风险？
- [ ] 优先使用已有依赖/stdlib，减少新增
- [ ] 依赖是否有已知安全漏洞？

---

## 审查结论模板

每次审查后，在答复中输出：

```
### 安全审查结论

| 项目 | 结果 |
|------|------|
| 数据安全 | ✅/⚠️/❌ |
| 系统稳定性 | ✅/⚠️/❌ |
| 运行负担 | ✅/⚠️/❌ |
| 依赖情况 | ✅/⚠️/❌ |

结论：可以安装 / 需要调整 / 不建议安装
```

---

## 审批门槛

| 审查结果 | 处理方式 |
|----------|----------|
| 全部 ✅ | 通知用户后直接安装 |
| 有 ⚠️ | 说明风险，等用户确认后再装 |
| 有 ❌ | 拒绝安装，给出替代方案 |
| 命中崩坏案例库相似致命模式 | 默认不继续尝试 |
