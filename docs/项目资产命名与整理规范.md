# 项目资产命名与整理规范

> 适用：材质 / 模型 / 地形 / 贴花的文件夹整理与命名标准化
> 决策优先级：文件不清晰 → 问点点。原则不清晰 → 以庞硕为准。

---

## 一、材质命名

### 1.1 通用规则

- 首字母大写，单词用 `_` 连接
- 同套贴图的不同颜色变体：加 `#red` / `#dark` 后缀
- 为列表直观，允许倒装：`Wall_Stone` 优于 `StoneWall`（同类材质在列表里聚在一起）

### 1.2 特殊不拆分规则

以下组合视为紧密词组，**内部不再加 `_`**：
- 数字 + 单位缩写：`H2m`、`2m`、`4m` 等
- 尺寸规格（数字 x 数字）：`3x3`、`2x4` 等

> 例如：`Wall_H2m_2m_x` — `H2m` 和 `2m` 内部不拆，`x` 是破损标识独立加。

### 1.3 精简化流程

从下载的原始材质名（如 `xxx网站-砖-尺寸-01-normal`）→ 精简为 `Brick01`，原始名称存入同名 txt。

### 1.3.1 批量重命名必须保留原名记录

每次执行批量重命名操作时，必须同时生成原名对照记录：

- **要求**：重命名脚本必须在执行时，将每一对「原名 → 新名」写入该目录下的 `rename_log.txt`
- **格式**：`原名  →  新名`（每行一条）
- **覆盖范围**：所有被改动的文件（含 .meta）
- **防漏机制**：脚本应先 dry-run 验证，确认无误后 execute，execute 时同步写入 log

> ⚠️ 此规则为事后补充（2026-06-03）。首次批量重命名 Materials_Standard 和 Terrain_Materials 时未记录原名，
> 现仅能通过后缀规则反向推导（见 `docs/Unity模型整理-材质重命名原名对照.md`）。

**文件夹内标准结构**：

```
Material_standard/
├── Mat_Brick01          （材质球）
├── Brick01_Col           （彩色图，统一用 _Col 替代 _BaseColor / _Albedo / _D / _A）
├── Brick01_Mask
├── Brick01_N
└── Brick01.txt           （原始贴图名字）
```

### 1.4 地形材质 CTL_

- 前缀：`CTL_`（当前暂用 `TL_`，后续统一为 `CTL_`）
- 地形贴图用于贴花时：`DECAL_` 前缀，贴图命名原则与材质贴图相同
- 地形材质文件夹内的 prefab → 移到 `Terrain materials/prefab/`

## 二、模型命名

### 2.1 非建筑模型

- 加归类前缀，material / model / prefab 名称统一（参考 `unit/` 文件夹范本）
- `Type-01` → 去掉 Type，直接 `Box_01`
- 精简版工程可能缺号（如只有 `xxx_05` 没有 1-4，因在大库中），本工程直接命名为 `xxx_01`，后续添加再续号

### 2.2 建筑模型

| 文件夹 | 说明 | 状态 |
|--------|------|------|
| `Building_Module` | 模块化建筑 | 待处理 |
| `Building_Standard` | 公司旧建筑资产 | 待精简 |
| `Building_unx` | 不可毁坏建筑 | 已升级，不动 |
| `Building_x` | 可毁坏建筑 | 已更新的移入 Building_Standard，遗留先不动 |

**Building_Standard 命名公式（精简后）**：

```
旧：House_类型_Medium_大小_l2_层数_uv_常用模型_02B1_型号_LODSta_FJL
新：House_类型_Medium_大小_l2_层数_常用模型_02B1_型号_#Brick_01
                                    ^^^^ 去掉 uv          ^^^^^^^^^^^ 去掉 LODSta_FJL
```

- 去掉 `uv`（以庞硕最终确认）
- 去掉 `LODSta` 和 `FJL`（制作人标识）
- 末尾加 `#材质名`：指向主要使用的材质（**因此材质必须先整理完**）

**破损/废墟标识**：
- `x`：破损版
- `xx`：废墟版
  > 注意：`x` 出现在尺寸类词组中（如 `3x3`）时不表破损，仅当后缀独立出现时表示破损。

---

## 三、执行顺序

1. **先整材质**（Material_standard）→ 材质名是建筑后缀依赖
2. **再整非建筑模型**（unit 等）
3. **最后整建筑模型**（Building_Standard / Building_Module）
