## 👑 deepseek-company/deepseek-v4-pro（主力）

- **ID**：`deepseek-company/deepseek-v4-pro`
- **用途**：当前会话主力模型，日常聊天 + 工作任务
- **Thinking**：支持 high/off
- **特点**：稳定、中文好、支持 reasoning
- **费用**：按 deepseek-company 定价

### 使用方式
- Web UI 下拉直接选
- 默认模型已设为它（`openclaw.json` → `models.defaultModel`）
- Cron 定时任务（早安/午安/晚安）也用它

### 已知坑点
无。当前最稳的主力模型。

## 🎨 litellm/agnes-image-2.1-flash（图生）

- **ID**：`litellm/agnes-image-2.1-flash`
- **用途**：文生图
- **API 网关**：LiteLLM → `https://apihub.agnes-ai.com/v1`
- **端点**：`/v1/images/generations`
- **支持尺寸**：1024x1024, 1024x1536, 1536x1024, 2048x2048
- **费用**：免费层

### 🔴 已知坑点（重要！）

#### 坑点 1：LiteLLM 管道返回格式异常
- **现象**：通过 `image_generate` 工具调用 `litellm/agnes-image-2.1-flash` 时，偶发 `LiteLLM image generation response malformed`
- **原因**：LiteLLM 管道对 Agnes 图生 API 的响应格式解析不稳定——URL 模式返回的图片链接可能被 LiteLLM 误判格式
- **触发率**：约 50%（下午 3 次成功，刚才 3 次失败）
- **绕过方法**：直接调 Agnes 原生 API

#### 绕过方案（直接调原生 API）
```bash
API_KEY=$(cat ~/.openclaw/openclaw.json | python3 -c \
  "import sys,json; print(json.load(sys.stdin)['models']['providers']['litellm']['apiKey'])")

# 生成图片
RESP=$(curl -s --max-time 120 \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  "https://apihub.agnes-ai.com/v1/images/generations" \
  -d '{
    "model": "agnes-image-2.1-flash",
    "prompt": "你的prompt",
    "n": 1,
    "size": "1024x1536"
  }')

# 提取 URL 并下载
URL=$(echo "$RESP" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['data'][0]['url'])")
wget -q -O /path/to/output.png "$URL"
```

#### 坑点 2：不支持 base64 响应格式
- `response_format: "b64_json"` → 报错 `UnsupportedParamsError`
- 只能走 URL 模式

#### 坑点 3：URL 图片下载可能超时
- `platform-outputs.agnes-ai.space` 从国内下载偶尔超时
- 用 `wget` 比 `curl` 稳定

## 🔮 nvidia/z-ai/glm-5.1（不稳定）

- **ID**：`nvidia/z-ai/glm-5.1`
- **用途**：备选聊天模型（NVIDIA API 上的 GLM-5.1）
- **状态**：⚠️ 不稳定，不推荐日常使用

### 🔴 已知坑点

#### 坑点 1：Thinking 不支持
- `thinkingLevel="high"` → `GatewayRequestError: thinkingLevel "high" is not supported`
- **必须**在切到这个模型前先把 thinking 关掉

#### 坑点 2：偶发 incomplete terminal response
- 即使 thinking=off，也会偶发 `incomplete terminal response`
- NVIDIA API 直连测试正常（`curl` 能返回），但 OpenClaw 流式解析偶尔断连
- 可能原因：OpenClaw 的 SSE/stream 解析与 GLM-5.1 响应格式不完全兼容

#### 坑点 3：不推荐日常使用
- 稳定性远不如 deepseek-company 系列
- 如果想尝鲜，建议只做短问答，不要切主会话
