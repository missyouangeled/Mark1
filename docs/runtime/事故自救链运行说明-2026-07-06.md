# 事故自救链运行说明（2026-07-06，2026-07-07 v2 更新）

## 一、目标与独立性

- 独立于现有保命层，专门处理“当前自己已经不可信”的事故场景
- 流程：留证 -> 选快照 -> 生成恢复计划 -> 隔离排障 -> 白名单自修 -> 验证
- 独立入口：`scripts/openclaw-incident-recovery.py`
- 独立状态目录：`~/.local/state/openclaw/incident-recovery/`
- 不复用 repair runner 的内部状态；不在坏状态下直接覆盖生产态

## 二、已纳入的事故模式（v2 / 2026-07-07）

### 1. `modelOverrideSource=auto` 的 fallback override 钉死
- `detect_auto_override_sessions()` 会扫 sessions.json，收集 modelOverrideSource=auto 的会话
- 输出：`autoOverrideRecovery`，区分 user override 与 auto override
- 现状：只输出安全清理方案，不直接改现网

### 2. `transcript repair` / `gateway restart` / `synthetic toolResult`
- `detect_transcript_repair_signals()` 结构化扫描主 session jsonl
- `real_events`（真实事件）与 `noise`（toolResult 文本回声）严格区分
- 真实事件识别字段：`assistant` 角色 + `provider=openclaw` + `model=gateway-injected`
  或 assistant 错误文本含 `inserted synthetic error result for transcript repair`
- 输出：`transcriptRepairRecovery.real_events` / `noise`

### 3. `reply session initialization conflicted for agent:main:main`
- `detect_reply_init_conflict_signals()` 同样结构化扫描
- 字段：assistant 文本含 `reply session initialization conflicted/failed`/`activeWork`，或 `kind=error` 事件含同样字面量
- 输出：`replyInitRecovery.severity`，含 `noise / degrading / activeWork_deadlock`

## 三、Severity 分级与运行策略

`classify_overall_severity()` 汇总 transcript repair 与 reply-init 冲突，输出四级：

| severity | 触发条件 | 策略 |
|---|---|---|
| `none` | 两条事故模式都没真实事件命中 | `observe`：前台轻交互继续 |
| `noise` | 只有 toolResult 文本回声，没有真实事件 | `observe_with_supervisor`：后台监工保持常驻，前台继续 |
| `degrading` | 真实事件 ≥1 条，但没形成持续性 deadlock | `background_supervisor_with_light_foreground` |
| `activeWork_deadlock` | transcript repair 与 reply-init 同时出现真实事件 | `isolate_and_split`：立即冻结主 transcript，重任务切到后台 subagent |

### 运行策略收敛：“后台执行 + 监工常驻 + 前台轻交互”

- **前台**：主会话只做触发 / 总结 / 审阅，不亲自跑长链 tool 调用、不读大文件
- **后台**：重任务（事故修复 / 重写 / 编译 / 大量 read）一律交给 `sessions_spawn`，明确 `taskName`、`promptFile`、`model`，独立上下文
- **监工**：incident-recovery / emergency-aggregator / health-collector / session-size-watcher / frontstage-guardian / repair-runner 持续采集证据，直到 severity 回到 noise / none 才退出
- **隔离恢复**：当 severity=activeWork_deadlock 时，主会话不再读故障 transcript，直接通过 isolated prompt 把任务交给 subagent；监工在后台继续判定是否升级

## 四、绝对边界（任何 severity 都不破）

- 不修改当前主 session 内容（jsonl / sessions.json）
- 不直接做 `openclaw sessions cleanup --enforce`
- 不直接做 `gateway restart`
- 不改 cron / systemd / gateway / openclaw 配置
- 不动 modelOverrideSource=`user` 的会话；auto override 也只输出清理方案不直接改

## 五、当前能力（截至 2026-07-07）

- 抓运行时证据（emergency / health / frontstage / openclaw status / system-summary / backup）
- 扫描 auto override / transcript repair（结构化） / reply-init（结构化） 真实事件与 noise
- 选择最近可用快照并附属于计划
- 汇总 severity 分级，决定是否切到 isolate_and_split
- 生成恢复计划（含 `severity` / `runtimeStrategy` / `transcriptRepairRecovery` / `replyInitRecovery`）
- 写入 isolation prompt，显式带上 severity 与运行策略
- 为后续自动 / 半自动恢复留接口（不直接动 gateway / cron / 配置）
