# 🔰 BOOT_INDEX.md — AI 启动入口

> **⚠️ 你是刚启动的 AI 模型。请严格按本文件的加载流程执行，不可跳过任何一步。**
>
> 本文件是「AI 需要读什么」的唯一信息源。以后新增需要读取的文件，只改这里即可。

---

## 触发条件

以下两种情况都必须执行本文件的完整加载流程：

1. **刚开机 / 新会话 / 新模型第一次接管当前会话**
2. **同一会话内切模型**（例如 Control UI 里只是改了当前会话模型，底层仍是同一个 session）
   - 判定方法：读取当前会话最近几条 `sessions_history`，找到最近一条 assistant 消息；如果那条消息的 `provider/model` 与当前运行模型 identity 不同，就视为“刚换模型”，也必须重走第 1-4 步。

## 加载流程（满足上面任一条件时，逐层执行）

### 第 1 步 — 读取 SOUL.md + USER.md（身份层）
- 路径：`SOUL.md`、`USER.md`
- 内容：我是谁、语言规则、对话风格、我在帮谁
- **读完立即用中文回复：「正在加载系统」（仅此一句，不要多说话）**
- 这条回复既适用于“新启动”，也适用于“同一会话内刚切模型后的第一条用户消息”

### 第 2 步 — 读取上下文层（今日状态）
- `memory/daily/YYYY-MM-DD.md`（今天 + 昨天）
- `/mnt/data/openclaw/session-backup/` 最新快照中的 `context-summary.md`
- 内容：今天发生了什么、最近规则变更、备用记忆

### 第 3 步 — 读取 MEMORY.md（偏好与规则层）
- 路径：`MEMORY.md`
- 内容：长期偏好、核心方法论、关系上下文、记忆归档索引

### 第 4 步 — 读取索引与技能层（全局导航）
- `SKILL_CATALOG.md` — 所有可用技能的目录
- `WORKSPACE_INDEX.md` — 工作区全局导航地图

### 第 5 步 — 全部加载完成，回复用户
- **在你的第一条回复最前面加上一行「OK 已经读取完成。」，然后正常回应用户的消息**
- 注意：第 1 步的「正在加载系统」已经由 chat.inject 自动注入（切模型场景），或由你自己发出（新启动场景）；这里只需确认加载完成即可
- 根据用户第一条消息的类型，从 `RULES_INDEX.md` 查表加载对应域规则
- 日常聊天 → `rules/chat.md`
- 工作任务 → `rules/work.md`
- 系统操作 → `rules/system.md` + `rules/work.md`
- 高风险 → 叠加 `rules/safety.md`
- 根据用户第一条消息的类型，从 `RULES_INDEX.md` 查表加载对应域规则

## 维护说明

- 新增 AI 必须读取的启动文件 → 更新本文件的第 1-4 步
- 新增按需加载文件 → 更新「按需加载」表格
- 修改启动顺序或回复行为 → 改本文件即可
- **不要分散修改 AGENTS.md / SOUL.md 的启动序列**

## 设计原则

- 启动时全量加载，不偷懒（防患于未然）
- Tier 1 读完立即回「正在加载系统」（让用户知道没卡死）
- 后续文件老实读完再开始正常对话
- 不要把“同一会话内切模型”误判为“没有启动事件就不用加载”；只要模型变了，就等价于一次新的接管
- 分层只是为了结构清晰，不是用来跳过加载的
