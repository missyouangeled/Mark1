# 🔰 BOOT_INDEX.md — AI 启动入口

> **⚠️ 你是刚启动的 AI 模型。请严格按本文件的加载流程执行，不可跳过任何一步。**
>
> 本文件是「AI 需要读什么」的唯一信息源。以后新增需要读取的文件，只改这里即可。

---

## 触发条件

以下两种情况都必须执行本文件的完整加载流程：

1. **刚开机 / 新会话 / 新模型第一次接管当前会话**
2. **同一会话内切模型**（例如 Control UI 里只是改了当前会话模型，底层仍是同一个 session）
   - 判定方法：读取当前会话最近几条 `sessions_history`，找到最近一条 assistant 消息；如果那条消息的 `provider/model` 与当前运行模型 identity 不同，就视为"刚换模型"，也必须重走第 1-5 步。

---

## 🔴 硬性输出要求（最高优先级）

无论上述哪种触发场景，**你在回复用户的第一个字之前**，必须先完成以下两项输出：

1. 读完第 1 步后 → 立即回复：**「正在加载系统」**（仅此一句）
   - 切模型场景：这句话已由前端 chat.inject 自动注入，你发完它即可
   - 新启动场景：你主动发出

2. 读完全部 0-5 步后 → 在你的第一条实际回复中，**第一行必须是「OK 已经读取完成。」**
   - 独占一行，然后空一行再回应用户的消息
   - 这是硬性规定，不能省略、不能替换措辞

## 加载流程

### 第 0 步 — 读取问题解决标准流程（前置）
- **任何模型启动后、处理问题前必须先读**
- 路径：`docs/通用-问题解决标准流程.md`
- 内容：六步法（搜索→分析→方案→自审→修复→验证）
- 即使不是系统操作类，只要涉及修改/排查，就必须读
- → 本步不计入「正在加载系统」的触发位置（仍在第 1 步后触发）

### 第 1 步 — 读取 SOUL.md + USER.md（身份层）
- 路径：`SOUL.md`、`USER.md`
- 内容：我是谁、语言规则、对话风格、我在帮谁
- → 回复「正在加载系统」

### 第 2 步 — 读取上下文层（今日状态）
- `memory/daily/YYYY-MM-DD.md`（今天 + 昨天）
- `/mnt/data/openclaw/session-backup/` 最新快照中的 `context-summary.md`

### 第 3 步 — 读取 MEMORY.md（偏好与规则层）
- 路径：`MEMORY.md`

### 第 4 步 — 读取索引与技能层（全局导航）
- `SKILL_CATALOG.md`
- `WORKSPACE_INDEX.md`

### 第 5 步 — 加载域规则，处理用户消息
- 根据用户第一条消息类型，从 `RULES_INDEX.md` 查表加载对应域规则
- 日常聊天 → `rules/chat.md` | 工作任务 → `rules/work.md` | 系统操作 → `rules/system.md` + `rules/work.md` | 高风险 → 叠加 `rules/safety.md`
- → **第一行输出「OK 已经读取完成。」**，然后回应用户

## 设计原则

- 启动时全量加载，不偷懒
- 分层只是为了结构清晰，不是用来跳过加载的
- 切模型 = 一次新的接管，必须完整加载
- 「OK 已经读取完成。」是硬指令，不是建议
