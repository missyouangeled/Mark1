# 多机器同步与维护 操作规则

> 按需加载：当涉及多机器同步、工作区更新、新机器注册时读取。
> 触发条件：用户说"同步这台机器" / "更新规则" / 首次在新机器启动

## 同步工作流

触发词："同步这台机器" / "更新这台机器" / "拉一下最新规则"

1. 到当前工作区 repo → `git pull --ff-only`
2. 若 repo 变了 → `openclaw gateway restart`
3. 重启后自动判定当前机器 → 读 `HOST_CONTEXT.md` → 匹配维护文档 → 自主核对

## 主机判定

优先级：runtime host 元数据 → OS hostname → 本地 IP 回退

## 新机器注册

- 未知设备 → 立即追加临时条目到 `HOST_CONTEXT.md`
- 命名：用户提供 > 硬件型号 > raw hostname
- 环境标签：证据强 → 公司/家里/掌机；不确定 → 未归类
- 告知用户分配的标签，可随时改名

## 规则隔离

- 主机标签只回答"哪台机器"，不改变规则集
- 共享规则跨设备一致
- 系统文档标注 `适用机器：公司（Linux）` / `掌机（Windows）`
- 检查顺序：`HOST_CONTEXT.md` → `docs/多机器-读取与更新规则.md` → 匹配机器的维护文档 → 对应脚本

## 掌机（Windows）特殊

- 拉取后默认读 Windows 文档，自主下载/安装/修复
- `.cmd` / `.ps1` 是手动回退入口
