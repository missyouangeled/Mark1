# 心跳/定时检查 操作规则

> 按需加载：当处理心跳 poll、定时任务、主动检查时读取。
> 触发条件：收到心跳 prompt、配置 cron job、用户问"主动检查什么"

## 心跳 vs Cron

- **心跳**：批处理多个检查（收件箱+日历+通知一轮搞定）、需要会话上下文、时间可漂移、减少 API 调用
- **Cron**：精确时间、独立于主会话、可指定模型/思维层级、一次性提醒、直投频道

## 心跳检查事项（轮转，每天 2-4 次）

- 邮件：有无紧急未读？
- 日历：未来 24-48h 事件？
- 提及：Twitter/社交通知？
- 天气：用户可能出门？

跟踪状态写 `memory/heartbeat-state.json`

## 何时出声

- 重要邮件到达
- 日历事件 <2h
- 发现有意思的东西
- 超过 8h 没说话

## 何时安静（HEARTBEAT_OK）

- 深夜 23:00-08:00（除非紧急）
- 用户在忙
- 无新内容
- 距上次检查 <30min

## 可自主做的事

- 读/整理记忆文件
- 检查项目（git status 等）
- 更新文档
- 提交推送自己的改动
- 审查更新 MEMORY.md

## 记忆维护（心跳期间）

每隔几天用一次心跳：
1. 读最近 `memory/daily/` 文件
2. 识别重要事件/教训
3. 更新 MEMORY.md 的提炼知识
4. 清理过时信息

> 每日文件是原始笔记；MEMORY.md 是精选智慧。
