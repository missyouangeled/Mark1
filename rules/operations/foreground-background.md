# 前台常驻 / 后台插播 操作规则

> 按需加载：当需要后台任务但前台必须保持响应时读取。
> 触发条件：复杂工程启动后台分身后

## 核心原则

- 主会话最高优先级，不能"像掉线一样"
- 默认不用 `sessions_yield` 结束主会话来"等后台结果"
- 前台继续正常对话，后台继续跑

## 进度报告

- 关键里程碑 / 阻塞点 / 最终完成 → 简短自然语言插播
- 后台分身误结束/空返回 → 前台先接住用户，再修正
- **3 分钟无可见产出** → 补进度说明
- 阻塞（重试/限流/验证码等）→ 主动说明卡点

## 负载卸载

- 重活尽早卸到后台分身
- 默认不把重活留在主会话同步执行
- 卸载后主会话立刻恢复可对话状态
