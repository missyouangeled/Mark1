# HOST_CONTEXT.md

把“这台机器是谁 / 在什么环境里”单独写清楚，避免每次都靠临场猜。

## 使用规则

- 优先使用运行时提供的 `host` 元数据判断当前机器。
- 如果没有运行时 `host`，再看系统 hostname / computer name。
- IP 只作为兜底标识，因为它可能会变化。
- 这里只定义**环境语义**，不是 GPS 或真实地理定位。
- 机器/地点标识只用于让 OpenClaw 知道“我现在在哪台机器上”，不用于把同一个仓库拆成两套不同规则。
- 只要各设备同步的是同一个仓库，默认应共享同一套行为规则。
- 除非用户明确另行指定，否则不得仅因“公司 / 掌机 / 家里”等标签不同，就切换 persona、权限、模型默认值、记忆策略或工作流。
- 未命中的机器不要强行猜地点；应按下面的“新机器自动登记规则”先写成临时条目。

## 新机器自动登记规则

- 只要当前机器没有命中下面任何已定义环境，就把它视为“新机器”。
- 新机器不要只在对话里口头说明，必须补写进本文件，避免下次再次失忆。
- 自动登记时至少写下：
  - `设备名`：给这台机器起一个人能看懂的临时名字
  - `环境标签`：高置信度才写 `公司` / `家里` / `掌机` 等；没把握就写 `未归类`
  - 匹配依据：运行时 `host`、系统 hostname / computer name、可用时再补稳定局域网 IP
  - `状态`：如果还是临时判断，标成“待确认”
- `设备名` 的生成优先级：
  1. 用户明确说过的名字
  2. 明显机型/设备角色（如 ROG 掌机、MacBook、NAS、工作机）
  3. 操作系统 + 设备类型（如 Windows 笔记本、Linux 机器、Mac）
  4. 最后才退回原始 hostname
- 新增条目时沿用下面已有环境的写法：一个标题 + 匹配条件 + 默认理解；如果环境标签还未确认，可直接把标题写成设备名，环境标签写 `未归类`。
- 自动登记只能新增未知设备或补充缺失匹配依据，不能覆盖用户已确认的设备名或环境标签。
- 用户后续如果说“这台是家里的”“这台叫书房那台”，应直接改写原条目，不要新增重复设备。

## 已定义环境

### 公司

满足以下任一条件，就把当前环境视为“公司”：

- 运行时主机名：`missyouangeled-VMware-Virtual-Platform`
- 系统 hostname / computer name：`missyouangeled-VMware-Virtual-Platform`
- 本地 IPv4（兜底）：`192.168.233.130`

匹配后默认采用这些理解：

- 设备名：`公司 Linux 机`
- 环境标签：`公司`
- 当用户问“现在这台是什么环境 / 是不是公司那台”时，应按“公司”理解
- 该标签仅用于识别当前机器身份，本身**不改变**共享规则

### 掌机

满足以下任一条件，就把当前环境视为“掌机”：

- 系统 hostname / computer name：`TABLET-EH5U3C01`

匹配后默认采用这些理解：

- 设备名：`ROG 掌机`
- 环境标签：`掌机`
- 当用户提到“掌机”“ROG 掌机”“那台平时带着走的设备”时，默认优先联想到这台机器
- 该标签仅用于识别当前机器身份，本身**不改变**共享规则
